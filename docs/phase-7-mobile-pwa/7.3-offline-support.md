# 7.3 — Offline Support

## What We're Building

Basic offline capability so the app doesn't break when the user loses internet. This is especially important for delivery boys who may be in areas with poor connectivity.

## Offline Strategy

**Full offline mode is NOT the goal.** The app requires internet for core operations (placing orders, changing status). But it should:
1. Load previously viewed data from cache
2. Show a clear "You're offline" indicator
3. Not crash or show blank screens
4. Sync actions when connectivity returns (limited scope)

## What Works Offline

| Feature | Offline Behavior |
|---------|-----------------|
| App shell (layout, navigation) | Cached by service worker — works |
| Product catalog | Cached from last fetch — viewable |
| Order history | Cached from last fetch — viewable |
| Delivery boy's active orders | Cached — viewable |
| Place new order | Blocked — show "You're offline" |
| Change order status | Queued — sync when online (optional) |
| Voice ordering | Blocked (Web Speech API needs internet) |

## Work Items

### 1. Service Worker Caching

The PWA module (from 7.1) handles static asset caching. For API data, configure Workbox strategies:

**Static assets** (JS, CSS, images): Cache First
- Serve from cache, update in background
- App shell loads instantly

**API data** (Firestore reads): Network First with Cache Fallback
- Try network first (get fresh data)
- If offline, serve from cache
- Firestore SDK has built-in offline persistence

### 2. Enable Firestore Offline Persistence

Firestore has built-in offline support. Enable it:

```typescript
import { enableIndexedDbPersistence } from 'firebase/firestore'

// In firebase plugin
try {
  await enableIndexedDbPersistence(db)
} catch (err) {
  if (err.code === 'failed-precondition') {
    // Multiple tabs open — persistence can only be enabled in one tab
  } else if (err.code === 'unimplemented') {
    // Browser doesn't support IndexedDB
  }
}
```

With this enabled:
- Firestore caches all documents read
- Queries work offline against the cache
- Writes are queued and sync when online

**Important**: This means the product catalog, order history, and delivery orders are all available offline automatically after the first load.

### 3. Offline Indicator Component

**File**: `components/common/OfflineIndicator.vue`

Shows a banner when the user is offline:

```
┌──────────────────────────────────────┐
│ ⚡ You're offline. Some features      │
│    may be unavailable.        [×]    │
└──────────────────────────────────────┘
```

Detection:

```typescript
const isOnline = ref(navigator.onLine)

window.addEventListener('online', () => { isOnline.value = true })
window.addEventListener('offline', () => { isOnline.value = false })
```

### 4. Disable Network-Required Actions

When offline, disable buttons that require network:
- "Place Order" button → disabled with tooltip "Requires internet"
- Voice mic button → disabled
- "Refresh" buttons → disabled

Don't hide them — just disable with a clear reason.

### 5. Offline Queue (Optional Enhancement)

For delivery boys, queuing status changes offline could be valuable:

1. Delivery boy marks "Delivered" while in a dead zone
2. Action is saved to a local queue (IndexedDB or Firestore offline queue)
3. When connectivity returns, Firestore syncs the write automatically

Since Firestore offline persistence handles this automatically (writes are queued), this may "just work" once persistence is enabled. Test it.

### 6. Cache Invalidation

When the app comes back online:
1. Firestore syncs automatically (pending writes sent, new data fetched)
2. Show a brief "Syncing..." indicator
3. Refresh active data (current page)

Don't force a full page reload — let Vue reactivity handle updates.

## Testing Offline

1. Chrome DevTools → Network → check "Offline"
2. Verify app shell loads
3. Navigate to products → data should show from cache
4. Try to place order → should be blocked
5. Uncheck "Offline" → app should recover

Also test on mobile:
1. Load the app on WiFi
2. Turn on airplane mode
3. Open the app from home screen
4. Verify cached content is visible
