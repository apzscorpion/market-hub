# 7.1 — PWA Configuration

## What We're Building

The configuration that makes the web app installable on mobile devices. After this, users can tap "Add to Home Screen" and the app looks and behaves like a native app.

## Work Items

npm install @vite-pwa/nuxt

### 2. Create App Icons

Generate icons in all required sizes. Start with a 512x512 source image and resize:

```
public/icons/
├── icon-72x72.png
├── icon-96x96.png
├── icon-128x128.png
├── icon-144x144.png
├── icon-152x152.png
├── icon-192x192.png
├── icon-384x384.png
├── icon-512x512.png
└── favicon.ico
```

The icon should be:
- Simple, recognizable at small sizes
- Relevant to wholesale/ordering (shopping bag, cart, or a custom logo)
- Good contrast on both light and dark backgrounds

Use a tool like `pwa-asset-generator` or create manually.

### 3. iOS-Specific Meta Tags

iOS (Safari) has its own PWA requirements. Add to `app.vue` or `nuxt.config.ts` head:

```html
<meta name="apple-mobile-web-app-capable" content="yes">
<meta name="apple-mobile-web-app-status-bar-style" content="black-translucent">
<meta name="apple-mobile-web-app-title" content="WDH">
<link rel="apple-touch-icon" href="/icons/icon-152x152.png">
```

### 4. Install Prompt

Show a custom "Install App" banner for users who haven't installed yet:

```typescript
// composables/useInstallPrompt.ts
function useInstallPrompt() {
  const deferredPrompt = ref<any>(null)
  const canInstall = ref(false)

  if (process.client) {
    window.addEventListener('beforeinstallprompt', (e) => {
      e.preventDefault()
      deferredPrompt.value = e
      canInstall.value = true
    })
  }

  async function install() {
    if (!deferredPrompt.value) return
    deferredPrompt.value.prompt()
    const result = await deferredPrompt.value.userChoice
    if (result.outcome === 'accepted') {
      canInstall.value = false
    }
  }

  return { canInstall, install }
}
```

Show a banner at the bottom of the screen:
```
┌──────────────────────────────────────┐
│ 📱 Install Wholesale Deal Hub        │
│    for faster access                 │
│    [Install]  [Not now]             │
└──────────────────────────────────────┘
```

### 5. Splash Screen

Configure the splash screen for when the app launches from the home screen:
- App name centered
- App icon
- Brand color background

This is handled automatically by the manifest's `background_color` and the icon. iOS may need additional `apple-touch-startup-image` links for proper splash screens.

### 6. Viewport and Mobile Optimizations

Ensure the app feels native on mobile:

```html
<meta name="viewport" content="width=device-width, initial-scale=1, maximum-scale=1, user-scalable=no">
```

- Disable pull-to-refresh (custom handling): `overscroll-behavior: none`
- Disable text selection on interactive elements: `user-select: none`
- Proper touch targets: minimum 44x44px for all buttons
- No horizontal scroll: `overflow-x: hidden` on body
