# 7.2 — Push Notifications

## What We're Building

Real push notifications that arrive on the user's phone even when the app is closed. Using Firebase Cloud Messaging (FCM) to send notifications triggered by order events.

## Notification Triggers

| Event | Who Gets Notified | Message |
|-------|------------------|---------|
| New order placed | Wholesaler | "New order from [Retailer] — [X items, ₹Y]" |
| Order accepted | Retailer | "Your order #[ID] has been accepted" |
| Order assigned to delivery | Delivery boy | "New delivery: [Retailer] at [Address]" |
| Order picked up | Retailer | "Your order is on the way" |
| Order delivered | Retailer + Wholesaler | "Order #[ID] delivered to [Retailer]" |
| Order cancelled | Retailer | "Your order #[ID] has been cancelled" |

## Architecture

Push notifications require server-side code (can't send from client). Using Firebase Cloud Functions:

```
Order status changes (Firestore trigger)
  → Cloud Function fires
    → Reads affected user(s)
    → Gets their FCM token(s)
    → Sends push notification via FCM
      → User's device receives notification
```

## Work Items

### 1. FCM Client Setup

**File**: `plugins/fcm.client.ts`

```typescript
import { getMessaging, getToken, onMessage } from 'firebase/messaging'

export default defineNuxtPlugin(async () => {
  const messaging = getMessaging()

  // Request notification permission
  const permission = await Notification.requestPermission()
  if (permission !== 'granted') return

  // Get FCM token
  const token = await getToken(messaging, {
    vapidKey: 'your-vapid-key-from-firebase-console'
  })

  // Save token to user's Firestore document
  // (so Cloud Functions know how to reach this device)

  // Handle foreground messages
  onMessage(messaging, (payload) => {
    // Show in-app notification
  })
})
```

### 2. Store FCM Token

When a user logs in and grants notification permission, save their FCM token to Firestore:

```
users/{userId}
├── ... existing fields
└── fcmTokens: string[]   ← array (user may have multiple devices)
```

Update the token on every app launch (tokens can expire/rotate).

### 4. Cloud Functions (Server-Side)

**File**: `functions/src/notifications.ts`

Firestore-triggered Cloud Functions that send notifications:

```typescript
import * as functions from 'firebase-functions'
import * as admin from 'firebase-admin'

export const onOrderStatusChange = functions.firestore
  .document('orders/{orderId}')
  .onUpdate(async (change, context) => {
    const before = change.before.data()
    const after = change.after.data()

    if (before.status === after.status) return // no status change

    // Determine who to notify based on the status change
    let targetUserId: string
    let title: string
    let body: string

    switch (after.status) {
      case 'accepted':
        targetUserId = after.retailerId
        title = 'Order Accepted'
        body = `Your order #${context.params.orderId.slice(0, 8)} has been accepted`
        break
      case 'shipped':
        targetUserId = after.retailerId
        title = 'Order Picked Up'
        body = 'Your order is on the way!'
        break
      // ... other cases
    }

    // Get user's FCM tokens
    const userDoc = await admin.firestore().doc(`users/${targetUserId}`).get()
    const tokens = userDoc.data()?.fcmTokens ?? []

    if (tokens.length === 0) return

    // Send notification
    await admin.messaging().sendEachForMulticast({
      tokens,
      notification: { title, body },
      data: { orderId: context.params.orderId, status: after.status },
    })
  })

export const onOrderCreated = functions.firestore
  .document('orders/{orderId}')
  .onCreate(async (snapshot, context) => {
    const order = snapshot.data()

    // Notify wholesaler(s)
    const wholesalerQuery = await admin.firestore()
      .collection('users')
      .where('role', '==', 'wholesaler')
      .get()

    for (const doc of wholesalerQuery.docs) {
      const tokens = doc.data().fcmTokens ?? []
      if (tokens.length === 0) continue

      await admin.messaging().sendEachForMulticast({
        tokens,
        notification: {
          title: 'New Order',
          body: `${order.retailerName} placed an order — ${order.items.length} items, ₹${order.totalAmount}`,
        },
        data: { orderId: context.params.orderId },
      })
    }
  })
```

### 5. Notification Permission UI

Don't ask for notification permission immediately on first visit (users will deny it).

Instead:
1. After the user's first successful order, show: "Want to know when your order is accepted? Enable notifications."
2. For delivery boys: ask on first login (notifications are essential for their workflow).
3. For wholesaler: ask on first login (need to know about new orders).

### 6. In-App Notification Handling

When a notification arrives while the app is open:
- Don't show a system notification (user is already in the app)
- Show an in-app toast at the top
- Update the notification bell count
- If on the relevant page (e.g., order detail), update the data in real-time

### 7. Notification Click Handling

When user taps a push notification:
- Open the app
- Navigate to the relevant page (e.g., order detail)
- Use the `data` payload to determine the route
