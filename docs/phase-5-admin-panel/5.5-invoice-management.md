## Features

### 1. Invoice Search & Filters

- Search by invoice number, retailer name, or phone
- Filter by status: All / Generated / Sent / Paid
- Filter by date range: Today / This Week / This Month / Custom
- Filter by retailer
- Sort by date, amount, status

### 2. Payment Tracking

Invoices have three statuses:
- **Generated**: Invoice created, not yet paid
- **Sent**: Invoice shared with retailer (optional intermediate state)
- **Paid**: Payment received

Mark as paid individually or in bulk (select multiple вҶ’ "Mark as Paid").

When marked as paid, record:
- `paidAt` timestamp
- Payment method (optional): cash / UPI / bank transfer

### 4. Bulk Print

Select multiple invoices вҶ’ "Print Selected" вҶ’ generates a printable page with each invoice on a separate print page (using CSS `page-break-after`).

Useful for wholesalers who hand physical invoices to delivery boys.

### 5. Export to CSV

Export filtered invoices to CSV for accounting:
- Invoice number
- Date
- Retailer name
- Items summary
- Subtotal, tax, total
- Status
- Paid date (if paid)

### 6. Invoice PDF Generation (Optional)

Generate downloadable PDF invoices. Options:

**Client-side**: Use `html2canvas` + `jsPDF` to convert the invoice HTML to PDF. Simple, no server needed.

**Server-side**: Use a Cloud Function with Puppeteer or a PDF template library. Better quality, consistent output.

For MVP, the printable HTML view is sufficient. PDF can wait.

### 7. Retailer Invoice Access

Retailers can view their invoices from the order detail page (already built in 1.8). Additionally, add an invoice list in the retailer's account:

**Page**: `/retailer/invoices`

Shows all invoices for the current retailer, with the ability to view and print each one.
