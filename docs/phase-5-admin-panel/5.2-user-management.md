## Work Items

### 1. Add User Flow

**Page**: `/wholesaler/users/new`

Form:
- Name (required)
- Phone number (required, must be valid Indian number, must be unique)
- Role: Retailer or Delivery (dropdown)
- Address (optional)

On submit:
1. Check if phone number already exists in `users` collection
2. If exists → show error "This phone number is already registered"
3. If new → create user document in Firestore
4. The user can now log in via phone OTP and will be recognized with their role

**The wholesaler creates the user document, but does NOT create the Firebase Auth account.** The Auth account is created when the user first logs in via OTP. The Firestore document (with role) is what authorizes them.

### 2. User Profile / Detail View

**Page**: `/wholesaler/users/[id]`

Shows:
- Name, phone, role
- Join date
- Account status (active/blocked)
- For retailers: total orders, total order value, last order date
- For delivery: total deliveries, pending deliveries
- Recent orders/deliveries list
- Block/Unblock button

### 3. Block/Unblock Users

Blocking a user:
1. Set `active: false` on their user document
2. The auth middleware checks `active` on every route — if false, redirect to "Account blocked" page
3. Log the action in activity logs

Unblocking reverses this.

**The blocked user's Firebase Auth session remains valid** (we can't invalidate it client-side). But since the app checks `active` from Firestore on every page load, they'll be locked out of the app immediately.

### 4. User Statistics

For each retailer, calculate and display:
- Total orders placed
- Total order value (sum of all order totals)
- Average order value
- Orders this month
- Last order date
- Most ordered products (top 5)

These are computed by querying the orders collection. For MVP, compute on-the-fly when viewing the profile. For performance at scale, pre-compute and cache.
