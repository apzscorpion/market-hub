# 5.6 — Stock Alerts & Sales Insights

## What We're Building

Business intelligence for the wholesaler: automated stock alerts that prevent stockouts, and sales analytics that reveal what's selling, who's buying, and how the business is trending.

## Part 1: Stock Alerts System

### Alert Types

| Alert | Trigger | Urgency |
|-------|---------|---------|
| **Low Stock** | Product quantity ≤ `lowStockThreshold` | Warning (yellow) |
| **Out of Stock** | Product quantity = 0 | Critical (red) |
| **Restocked** | Product quantity goes from 0 → positive | Info (green) |
| **High Demand** | Product ordered more than X times today | Info (blue) |

### Alert Delivery

1. **Dashboard Widget** (always visible): Low stock count badge on sidebar, alerts panel on dashboard
2. **Push Notification** (Phase 7): "Vanilla Big Pack is out of stock — 3 pending orders need it"
3. **Daily Summary** (optional): Morning notification summarizing stock status

### Smart Alert: Orders Affected

When a product is out of stock, the alert shows how many pending/accepted orders contain that product. This creates urgency — the wholesaler knows they need to either restock or contact those retailers.

```typescript
async function getAffectedOrders(productId: string): Promise<number> {
  const orders = await getDocs(
    query(
      collection(db, 'orders'),
      where('status', 'in', ['pending', 'accepted']),
    )
  )
  return orders.docs.filter(doc =>
    doc.data().items.some((item: any) => item.productId === productId)
  ).length
}
```

### Metrics to Track

**Revenue Metrics**:
- Total revenue (this period)
- Revenue growth (month-over-month)
- Average order value
- Revenue by product
- Revenue by retailer

**Order Metrics**:
- Total orders (this period)
- Orders per day (trend line)
- Orders by status (completion rate)
- Orders by source (manual vs voice)
- Cancellation rate

**Product Metrics**:
- Top selling products (by quantity and revenue)
- Fast-moving items (velocity — units/day)
- Slow-moving items (not ordered in X days)
- Products never ordered (dead inventory)
- Category performance

**Retailer Metrics**:
- Top retailers by order count and value
- New vs returning retailers
- Average order frequency per retailer
- Retailers who stopped ordering (churn risk)

### Pre-Computed vs Real-Time

Computing analytics by querying all orders every time is expensive. Strategy:

**Pre-computed** (recommended for most metrics):
- Use a Cloud Function that runs daily (or on order events)
- Writes summary data to an `analytics` collection
- Dashboard reads the pre-computed summary (1 read)

**Real-time** (for "today's" data):
- Query today's orders directly
- Small dataset, acceptable cost

```
analytics/monthly-2026-04
├── revenue: number
├── orderCount: number
├── avgOrderValue: number
├── topProducts: [{ productId, name, quantity, revenue }]
├── topRetailers: [{ userId, name, orderCount, revenue }]
├── ordersBySource: { manual: number, voice: number }
└── updatedAt: timestamp
```

Charts are nice but not essential. Numeric summaries are more actionable for a busy wholesaler.
