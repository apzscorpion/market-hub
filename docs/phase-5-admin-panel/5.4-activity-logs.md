# 5.4 — Activity Logs & Analytics

## What We're Building

An audit trail of all significant actions in the system, plus a basic analytics dashboard for business insights.

## Activity Logs

### What Gets Logged

| Action | Trigger | Who |
|--------|---------|-----|
| `order_created` | Retailer places order | Retailer |
| `order_status_changed` | Status updated | Wholesaler / Delivery |
| `order_items_edited` | Order items modified | Wholesaler |
| `order_cancelled` | Order cancelled | Wholesaler |
| `product_created` | New product added | Wholesaler |
| `product_updated` | Product details changed | Wholesaler |
| `product_deleted` | Product removed | Wholesaler |
| `products_imported` | Bulk CSV import | Wholesaler |
| `user_created` | New user added | Wholesaler |
| `user_blocked` | User blocked | Wholesaler |
| `user_unblocked` | User unblocked | Wholesaler |
| `voice_order` | Voice ordering used | Retailer |

### Log Entry Structure

```typescript
interface ActivityLog {
  id: string
  action: string
  performedBy: string          // userId
  performedByName: string      // denormalized
  performedByRole: string
  targetType: 'order' | 'product' | 'user' | 'system'
  targetId: string
  details: Record<string, any> // action-specific data
  createdAt: Date
}
```

Example details for `order_status_changed`:
```json
{
  "previousStatus": "pending",
  "newStatus": "accepted",
  "orderId": "abc123",
  "retailerName": "Rajan's Store"
}
```

### Logging Utility

**File**: `utils/activityLogger.ts`

```typescript
async function logActivity(
  action: string,
  targetType: string,
  targetId: string,
  details: Record<string, any>
): Promise<void> {
  const authStore = useAuthStore()
  await addDoc(collection(db, 'activityLogs'), {
    action,
    performedBy: authStore.user?.id,
    performedByName: authStore.user?.name,
    performedByRole: authStore.user?.role,
    targetType,
    targetId,
    details,
    createdAt: serverTimestamp(),
  })
}
```

Call this function from every composable that modifies data. It's fire-and-forget — don't await it in the user-facing flow (log failure shouldn't block the action).

### Voice Session Logs

Special section for reviewing voice ordering sessions:

**Page**: `/wholesaler/logs/voice`

Shows:
- Each voice session with raw transcript
- What products were matched
- What products were NOT matched
- User corrections (what the user changed after voice)

This data drives alias improvement:
- If "vanela" is frequently spoken but not matched → add it as alias for Vanilla
- If a product is never found by voice → its alias list needs work

### Analytics Composable

**File**: `composables/useAnalytics.ts`

Functions:
- `getOrderStats(dateRange)` — count, total value, average value
- `getTopProducts(limit)` — most ordered products
- `getRetailerStats()` — orders per retailer
- `getVoiceUsageStats()` — % of orders via voice vs manual

All computed from Firestore queries on the orders collection.
