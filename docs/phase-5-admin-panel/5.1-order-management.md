# 5.1 — Advanced Order Management

## What We're Building

Upgrade the wholesaler's order handling from basic status changes (Phase 1) to full order control: edit items, add notes, bulk status changes, and order printing.

## New Order Capabilities

### 1. Edit Order Items

The wholesaler should be able to modify an order's items. Common reasons:
- Product out of stock → reduce quantity or remove
- Retailer called to add something → add item
- Wrong product → swap for the right one

**Editable fields on an order** (for wholesaler only):
- Item quantities (change any qty)
- Remove items
- Add new items (same autocomplete as retailer's order table)
- Add/edit order notes

**Rules**:
- Only editable when status is `pending` or `accepted`
- Shipped and delivered orders are locked
- Every edit is logged in `statusHistory` with what changed

**UI**: The order detail page gets an "Edit" mode toggle. In edit mode, the items table becomes editable (similar to the retailer's order table from Phase 2).

### 2. Bulk Status Change

Select multiple orders from the list and change their status at once:
- Select checkbox on each order card/row
- "Change Status" dropdown appears in toolbar
- Select new status → confirmation dialog → apply to all selected

Use case: Wholesaler accepts 10 pending orders at once at the start of the day.

### 3. Order Notes

Two types of notes:

**Retailer notes** (already exists from Phase 1): The retailer adds a note when placing the order ("Please deliver before 10 AM").

**Wholesaler notes** (new): Internal notes the wholesaler adds. Not visible to the retailer. Used for tracking ("Called retailer, confirmed quantities").

```
order.notes: string          // retailer notes (existing)
order.adminNotes: string     // wholesaler internal notes (new)
```

### 4. Order Printing

Generate a printable order summary:
- Order ID and date
- Retailer name and phone
- Items table (product, qty, unit, price, total)
- Grand total
- Status
- Notes

Use CSS `@media print` to create a clean print layout. No separate PDF generation needed — browser print is sufficient.

**File**: `components/wholesaler/OrderPrint.vue`

### 5. Order Search & Advanced Filters

Upgrade the order list:
- Search by retailer name or phone
- Search by product name in order (find all orders containing "Vanilla")
- Date range picker (custom dates, not just Today/Week/Month)
- Export filtered orders to CSV

### 6. Real-Time Order Updates

Replace the manual refresh with Firestore real-time listeners:
- Dashboard auto-updates when new orders arrive
- Status changes from delivery boys appear instantly
- Use `onSnapshot` on the orders collection

Show a notification when a new order arrives: "New order from Rajan's Store"
