# 4.4 — Image Management

## What We're Building

The ability to upload, store, and display product images. Products will show real photos instead of placeholder icons.

## Image Upload Flow

```
1. Wholesaler opens product form (new or edit)
2. Clicks "Add Image" area
3. File picker opens (or camera on mobile)
4. Selects image
5. Image is compressed client-side
6. Compressed image uploads to Firebase Storage
7. Download URL is saved to the product document
8. Image thumbnail shows in the form
```

### 2. Client-Side Image Compression

Before uploading, compress images to save storage and bandwidth:

```typescript
async function compressImage(file: File, maxWidth = 800, quality = 0.8): Promise<Blob> {
  const bitmap = await createImageBitmap(file)
  const canvas = document.createElement('canvas')

  let width = bitmap.width
  let height = bitmap.height

  if (width > maxWidth) {
    height = (height / width) * maxWidth
    width = maxWidth
  }

  canvas.width = width
  canvas.height = height

  const ctx = canvas.getContext('2d')!
  ctx.drawImage(bitmap, 0, 0, width, height)

  return new Promise(resolve => {
    canvas.toBlob(blob => resolve(blob!), 'image/jpeg', quality)
  })
}
```

Target: max 800px wide, JPEG quality 80%. This keeps product images under 100KB typically.

### 3. Firebase Storage Upload

**File**: `composables/useImageUpload.ts`

```typescript
function useImageUpload() {
  const progress = ref(0)
  const uploading = ref(false)
  const error = ref<string | null>(null)

  async function uploadProductImage(productId: string, file: File): Promise<string> {
    uploading.value = true
    progress.value = 0
    error.value = null

    const compressed = await compressImage(file)
    const path = `products/${productId}/${Date.now()}.jpg`
    const storageRef = ref(storage, path)

    const uploadTask = uploadBytesResumable(storageRef, compressed)

    return new Promise((resolve, reject) => {
      uploadTask.on('state_changed',
        (snapshot) => {
          progress.value = (snapshot.bytesTransferred / snapshot.totalBytes) * 100
        },
        (err) => {
          error.value = err.message
          uploading.value = false
          reject(err)
        },
        async () => {
          const url = await getDownloadURL(uploadTask.snapshot.ref)
          uploading.value = false
          resolve(url)
        }
      )
    })
  }

  async function deleteImage(imageUrl: string): Promise<void> {
    const storageRef = ref(storage, imageUrl)
    await deleteObject(storageRef)
  }

  return { uploadProductImage, deleteImage, progress, uploading, error }
}
```

### 4. Image Display

Product images display in two contexts:

**Product card (retailer)**:
- First image as thumbnail (150×150px, object-cover)
- If no image: show a colored placeholder with the first letter of the product name

**Product detail/form (wholesaler)**:
- All images shown in a row
- Click to enlarge (lightbox)
- First image is marked as "Primary"

### 6. Image Cleanup

When a product is deleted, its images should also be deleted from Storage. The `deleteProduct` function should:
1. Read the product's `images` array
2. Delete each image from Storage
3. Delete the product document from Firestore

If image deletion fails, log the error but still delete the product. Orphaned images can be cleaned up later.

## Image Optimization for Display

For the retailer's product grid (lots of thumbnails), performance matters:

- Use `loading="lazy"` on `<img>` tags
- Set explicit `width` and `height` to prevent layout shift
- Use CSS `object-fit: cover` for consistent card sizes
- Consider generating smaller thumbnails (200px) for list views vs. full size for detail views (future optimization)

## Storage Limits

Firebase Storage free tier: 5GB storage, 1GB/day download.

With compressed images averaging 80KB each:
- 5GB = ~62,500 images
- 100 products × 3 images = 300 images = 24MB

Free tier is more than sufficient for a long time.
