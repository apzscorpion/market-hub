# 4.1 — Category Structure

## What We're Building

A two-level hierarchy for organizing products: **Category → Subcategory → Products**. This makes browsing manageable when the catalog grows.

## Example Structure

```
Ice Cream
├── Cones
│   ├── Vanilla Cone
│   ├── Chocolate Cone
│   └── Butterscotch Cone
├── Bars
│   ├── Choco Bar Classic
│   ├── Choco Bar Premium
│   └── Mango Bar
├── Cups
│   ├── Vanilla Cup
│   └── Strawberry Cup
└── Family Packs
    ├── Vanilla 1L
    └── Chocolate 1L

Frozen Snacks
├── Kulfi
│   ├── Malai Kulfi
│   └── Pista Kulfi
└── Popsicles
    ├── Orange Pop
    └── Cola Pop
```

## Data Model (Already Defined in Phase 0)

Categories are stored in Firestore with subcategories embedded:

```
categories/{categoryId}
├── name: "Ice Cream"
├── nameML: "ഐസ്ക്രീം"
├── sortOrder: 1
├── subcategories: [
│   { id: "cones", name: "Cones", nameML: "കോൺ", sortOrder: 1 },
│   { id: "bars", name: "Bars", nameML: "ബാർ", sortOrder: 2 },
│   ...
│ ]
```

Products reference `categoryId` and `subcategoryId`.

## Work Items

### 1. Category Management UI (Wholesaler)

**Page**: `/wholesaler/categories`

Features:
- List all categories with their subcategories
- Add new category (name + Malayalam name)
- Edit category name
- Delete category (only if no products use it)
- Reorder categories (drag-drop or sort number)
- Add/edit/delete subcategories within a category

### 2. Category Composable

**File**: `composables/useCategories.ts`

Functions:
- `fetchCategories()` — load all categories
- `addCategory(name, nameML)` — create new
- `updateCategory(id, data)` — edit
- `deleteCategory(id)` — delete (with products check)
- `addSubcategory(categoryId, subcategory)` — add sub
- `removeSubcategory(categoryId, subcategoryId)` — remove sub
- `reorderCategories(orderedIds)` — update sort orders

### 3. Category Tabs (Retailer)

Update the retailer product page to show category tabs at the top. Tapping a tab filters products to that category. "All" tab shows everything.

### 4. Category Selector (Product Form)

When adding/editing a product, the category and subcategory are selected via cascading dropdowns:
1. First dropdown: select category
2. Second dropdown: populated with subcategories of the selected category

## Why Two Levels Only

Three+ levels add complexity without value for this use case. A wholesaler's catalog is typically:
- 5-10 categories
- 2-5 subcategories per category
- 5-20 products per subcategory

Two levels is enough. If it's not, the naming convention handles it ("Premium Vanilla Cone 500ml" is descriptive enough).
