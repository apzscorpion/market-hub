# 4.3 — Search & Filters

## What We're Building

Enhanced search and filtering for both the retailer product catalog and the wholesaler product management page. Making it fast to find any product in a growing catalog.

## Retailer Search (Product Catalog)

### Current State (Phase 1)
Basic text search against product names.

### Upgraded Search
- Searches across: name, nickname, aliases (all three)
- Results show which field matched (e.g., "matched alias: വാനില")
- Debounced input (150ms) for smooth typing
- Search highlights the matching text in results
- Malayalam and English queries both work
- Empty search shows all products (with category filter applied)

### Category Filters
- Horizontal tab bar at the top: [All] [Cones] [Bars] [Cups] ...
- Tapping a category filters immediately (client-side)
- Active category tab is highlighted
- Count badge on each tab showing number of products

### Combined Search + Filter
- Category filter AND search work together
- If "Cones" tab is active and user searches "vanilla" → shows only vanilla cones
- Clear search restores full category view

### Sort Options
- Default: by sort order (admin-defined)
- Price: low to high
- Price: high to low
- Name: A to Z
- Name: Z to A

Small dropdown at top-right to change sort.

## Wholesaler Search (Product Management)

### Enhanced Product Table Search
The wholesaler product table gets:
- Same text search as retailer (name, nickname, aliases)
- Status filter: All / Active / Inactive
- Category filter: dropdown (not tabs, since it's a table)
- Date range: sort by recently added

### Quick Filters
Preset buttons above the table:
- "All Products" (default)
- "Active Only"
- "Inactive Only"
- "No Image" (products missing images — useful for cleanup)
- "No Aliases" (products without voice aliases — useful for voice prep)

## Implementation

### Search Utility

**File**: `utils/productSearch.ts`

Reusable search function shared between retailer and wholesaler views:

```typescript
interface SearchOptions {
  query: string
  categoryId?: string
  subcategoryId?: string
  activeOnly?: boolean
  sortBy?: 'name' | 'price-asc' | 'price-desc' | 'sortOrder'
}

function filterProducts(products: Product[], options: SearchOptions): Product[] {
  let filtered = [...products]

  // Category filter
  if (options.categoryId) {
    filtered = filtered.filter(p => p.categoryId === options.categoryId)
  }
  if (options.subcategoryId) {
    filtered = filtered.filter(p => p.subcategoryId === options.subcategoryId)
  }

  // Active filter
  if (options.activeOnly) {
    filtered = filtered.filter(p => p.active)
  }

  // Text search
  if (options.query) {
    const q = options.query.toLowerCase().normalize('NFC')
    filtered = filtered.filter(p =>
      p.name.toLowerCase().includes(q) ||
      p.nickname.toLowerCase().includes(q) ||
      p.aliases.some(a => a.toLowerCase().includes(q))
    )
  }

  // Sort
  switch (options.sortBy) {
    case 'name':
      filtered.sort((a, b) => a.name.localeCompare(b.name))
      break
    case 'price-asc':
      filtered.sort((a, b) => a.price - b.price)
      break
    case 'price-desc':
      filtered.sort((a, b) => b.price - a.price)
      break
    case 'sortOrder':
    default:
      filtered.sort((a, b) => a.sortOrder - b.sortOrder)
  }

  return filtered
}
```

### Search Highlight

When displaying search results, highlight the matching portion of text:

```typescript
function highlightMatch(text: string, query: string): string {
  if (!query) return text
  const regex = new RegExp(`(${escapeRegex(query)})`, 'gi')
  return text.replace(regex, '<mark>$1</mark>')
}
```

Render with `v-html` (safe because the query comes from the user's own input, not from other users).
