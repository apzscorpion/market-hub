# 4.2 — Product CRUD

## What We're Building

The wholesaler's interface for adding, editing, and deleting products. This replaces the manual Firestore seeding from earlier phases.

## Product Form

**Page**: `/wholesaler/products/new` (create) and `/wholesaler/products/[id]/edit` (edit)

### Form Fields

| Field | Input Type | Required | Notes |
|-------|-----------|----------|-------|
| Name | Text input | Yes | Full product name |
| Nickname | Text input | Yes | Short name for display and search |
| Category | Dropdown | Yes | From categories collection |
| Subcategory | Dropdown | Yes | Filtered by selected category |
| Unit Type | Dropdown | Yes | box / piece / pack / kg / litre |
| Price | Number input | Yes | In rupees, minimum 0 |
| Aliases | Tag input | No | Multiple values, press Enter to add each |
| Images | File upload | No | Multiple images, first is primary |
| Active | Toggle | Yes | Default true. False hides from catalog. |
| Sort Order | Number | No | For manual ordering within category |

## Work Items

### 1. Product List Page (Wholesaler)

**Page**: `/wholesaler/products`

A table listing all products:

| Column | Content |
|--------|---------|
| Image | Thumbnail (first image) |
| Name | Full product name |
| Nickname | Short name |
| Category | Category > Subcategory |
| Price | ₹ formatted |
| Status | Active / Inactive badge |
| Actions | Edit / Delete buttons |

Features:
- Search by name/nickname
- Filter by category
- Filter by active/inactive
- Sort by name, price, or date added
- Pagination (if > 50 products)

### 2. Product Form Component

**File**: `components/wholesaler/ProductForm.vue`

Reused for both create and edit. Receives optional `product` prop for edit mode.

Validation:
- Name and nickname required, min 2 characters
- Category and subcategory required
- Price must be >= 0
- Unit type must be selected
- Show validation errors inline below each field

### 3. Product Composable (Extended)

**File**: `composables/useProducts.ts` (modify)

Add functions:
- `createProduct(data)` — write to Firestore
- `updateProduct(id, data)` — update document
- `deleteProduct(id)` — delete (with confirmation)
- `toggleProductActive(id)` — quick toggle active status

### 4. Alias Tag Input Component

**File**: `components/common/TagInput.vue`

Reusable tag input:
- Text field with "Add" button or Enter to add
- Each tag shows with × to remove
- Duplicate prevention
- No empty tags

### 5. Delete Confirmation

Deleting a product should:
- Show confirmation dialog
- Check if product is referenced in any pending/accepted orders
- If referenced: warn "This product is in X active orders. Deactivate instead?"
- If not referenced: delete the document

### 6. Bulk Actions (Basic)

Select multiple products via checkboxes for:
- Bulk activate / deactivate
- Bulk delete (with confirmation)

Full bulk operations (CSV import/export) come in Phase 5.
