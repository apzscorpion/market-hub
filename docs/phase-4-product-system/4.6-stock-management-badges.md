# 4.6 вҖ” Stock Management & Product Badges

## What We're Building

Two features that directly drive business:
1. **Stock Management** вҖ” Track what's available, alert when running low, show stock status to retailers
2. **Product Badges** вҖ” Flag products as New, Trending, or Recommended to promote discovery

## Part 1: Stock Management

### Why Stock Matters

Without stock visibility:
- Retailers order products that are out of stock вҶ’ wholesaler has to call them back to modify вҶ’ wastes everyone's time
- Wholesaler doesn't know when to restock until they physically check the warehouse
- Orders for unavailable items create frustration

With stock visibility:
- Retailers see "In Stock" / "Out of Stock" / "Low Stock" before ordering
- Wholesaler gets alerts before running out
- Orders only contain available products

### Stock Data Model

Already defined in the data model (0.2):

```typescript
stock: {
  quantity: number           // current count in units (boxes, pieces, etc.)
  lowStockThreshold: number  // alert when quantity drops below this
  lastUpdated: timestamp
}
```

### Stock Update Methods

**Manual Update** (MVP):
- Wholesaler goes to product edit page вҶ’ updates stock quantity
- Wholesaler uses a dedicated "Stock Update" page for quick batch updates

**On Order Accept** (Automatic):
- When an order is accepted, reduce stock for each item:
  ```
  stock.quantity -= orderItem.quantity
  ```
- If stock goes below 0, set to 0 and flag as out of stock

**On Order Cancel** (Automatic):
- If an accepted order is cancelled, restore stock:
  ```
  stock.quantity += orderItem.quantity
  ```

### Order Validation Against Stock

When placing an order, check stock:

1. **Soft check** (client-side, on add to table): Warn if quantity exceeds stock
2. **Hard check** (on submit): Verify stock in Firestore before writing order
3. If stock insufficient at submit time: "Vanilla Big has only 3 left. You requested 5. Reduce to 3?"

This prevents over-ordering without blocking the flow.

### Auto-Expire "New" Badge

Products shouldn't stay "New" forever. Auto-expire based on system setting:

```typescript
function isNew(product: Product): boolean {
  if (!product.badges.isNew) return false

  const daysOld = daysSince(product.createdAt)
  const maxDays = systemSettings.newBadgeDurationDays // default 14

  return daysOld <= maxDays
}
```

After 14 days, the "New" badge disappears automatically. Wholesaler can also manually toggle it off earlier or keep it on.
