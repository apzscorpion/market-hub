# 4.5 — Product Classification System

## What We're Building

A 5-dimensional classification system that makes every product precisely categorizable. Instead of just "Category → Product", each product is classified by Type, Flavor, Variant, Packaging, and Temperature Sensitivity.

## Why 5 Dimensions

A single category/subcategory hierarchy is too flat for a product catalog with real variety:

- "Vanilla Cone" and "Vanilla Family Pack" are both vanilla, but completely different products
- "Choco Bar Premium" and "Choco Bar Kids" are both bars, but different variants
- A retailer searching by "type: cone" wants ALL cones regardless of flavor
- A retailer searching by "flavor: vanilla" wants ALL vanilla regardless of type

Multi-dimensional classification enables any combination of filters.

## Implementation

### Classification Config Document

Store all classification options in a single Firestore document:

**Document**: `classificationOptions/config`

```typescript
interface ClassificationConfig {
  types: ClassificationOption[]
  flavors: ClassificationOption[]
  variants: ClassificationOption[]
  packagingTypes: ClassificationOption[]
  temperatureLevels: ClassificationOption[]
}

interface ClassificationOption {
  id: string
  name: string
  nameML: string
  sortOrder: number
  active: boolean
}
```

One document = one read to get all options. Cached after first load.

### Filter UI (Retailer)

The retailer product page gets multi-dimensional filters:

```
┌──────────────────────────────────────────────┐
│  Products                    🔍 Search       │
├──────────────────────────────────────────────┤
│  Type: [All] [Cone] [Bar] [Cup] [Pack] ...  │
│  Flavor: [All] [Vanilla] [Choco] [Mango]... │
│  [✓ Show only available]                    │
├──────────────────────────────────────────────┤
│  (filtered product grid)                     │
└──────────────────────────────────────────────┘
```

Filters are combinable: "Type: Cone" + "Flavor: Vanilla" shows only vanilla cones.

### Admin Management

The wholesaler can manage classification options:

**Page**: `/wholesaler/settings/classification`

- Add/edit/remove types, flavors, variants
- Reorder options
- Add Malayalam translations
- Deactivate options (products using them keep the value, but it's hidden from new product forms)

## Auto-Generated Aliases from Classification

When a product's classification is set, auto-generate aliases from the classification:

```typescript
function generateClassificationAliases(product: Product): string[] {
  const aliases: string[] = []
  const c = product.classification

  // Type + Flavor combos
  aliases.push(`${c.flavor} ${c.typeId}`)       // "vanilla cone"
  aliases.push(`${c.typeId} ${c.flavor}`)        // "cone vanilla"

  // Malayalam equivalents (from config)
  // ... lookup nameML for each dimension and combine

  return aliases
}
```

These auto-aliases supplement the manually entered ones, making voice matching richer without extra work.
