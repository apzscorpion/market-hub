# 8.1 — Image Pipeline

## What We're Building

A polished image system that handles thumbnails, image galleries, bulk uploads, and optimized delivery. Building on the basic upload from Phase 4.

## Improvements Over Phase 4

### 1. Thumbnail Generation

Phase 4 uploads one size. Now generate multiple sizes:

| Size | Dimensions | Use Case |
|------|-----------|----------|
| Thumbnail | 200×200 | Product grid cards |
| Medium | 400×400 | Product detail view |
| Original | Max 800px wide | Full-size lightbox |

Generate on upload (client-side):

```typescript
async function generateSizes(file: File): Promise<{
  thumbnail: Blob
  medium: Blob
  original: Blob
}> {
  return {
    thumbnail: await compressImage(file, 200, 0.7),
    medium: await compressImage(file, 400, 0.8),
    original: await compressImage(file, 800, 0.85),
  }
}
```

Store in Firebase Storage:
```
products/{productId}/
├── thumb_1713020400000.jpg
├── med_1713020400000.jpg
└── orig_1713020400000.jpg
```

Product document stores URLs for each size:
```typescript
interface ProductImage {
  thumbnailUrl: string
  mediumUrl: string
  originalUrl: string
}
```

### 2. Image Gallery Component

**File**: `components/common/ImageGallery.vue`

For viewing product images:
- Swipe left/right on mobile
- Arrow navigation on desktop
- Pinch to zoom
- Lightbox overlay for full-size view
- Image counter: "2 / 5"

Keep it simple — use a lightweight library like `vue-easy-lightbox` or build a minimal version.

### 3. Bulk Image Upload

**File**: `components/wholesaler/BulkImageUpload.vue`

For adding images to multiple products quickly:

1. Wholesaler selects multiple files (or drags a batch)
2. Each file shows as a card with a preview
3. For each image, wholesaler selects the product it belongs to (autocomplete)
4. Click "Upload All"
5. Progress bar per image
6. Results: "8/10 uploaded. 2 failed."

Alternative flow: Name the files to match products (e.g., `vanilla-big.jpg`) and auto-match by name.

### 4. Image Optimization for Bandwidth

**Lazy loading**: All images use `loading="lazy"` to defer off-screen images.

**Progressive loading**: Show a tiny blurred placeholder while the real image loads (LQIP — Low Quality Image Placeholder):
1. On upload, generate a 10×10px version → base64 encode → store inline in product document
2. Display the blurred placeholder immediately
3. When the real image loads, crossfade

```typescript
interface ProductImage {
  thumbnailUrl: string
  mediumUrl: string
  originalUrl: string
  placeholder: string  // base64 of 10x10 blurred version
}
```

### 5. Image Cache Headers

Firebase Storage serves files with configurable cache headers. Set:
- `Cache-Control: public, max-age=31536000` (1 year)
- Images are immutable (new upload = new filename) so long cache is safe

```typescript
const metadata = {
  cacheControl: 'public, max-age=31536000',
  contentType: 'image/jpeg',
}
await uploadBytes(storageRef, compressed, metadata)
```

### 6. WebP Conversion (Optional)

Convert uploaded images to WebP format for 25-35% smaller file sizes:

```typescript
canvas.toBlob(blob => resolve(blob!), 'image/webp', 0.8)
```

Check browser support:
```typescript
const supportsWebP = document.createElement('canvas').toDataURL('image/webp').startsWith('data:image/webp')
```

Fall back to JPEG if WebP is not supported.
