# 9.4 — CI/CD & Deployment

## What We're Building

Automated deployment pipeline. Push code to GitHub → tests run → app deploys to staging or production. No manual `firebase deploy` commands.

## Deployment Environments

| Environment | Branch | URL | Purpose |
|-------------|--------|-----|---------|
| Development | Local | `localhost:3000` | Active development |
| Staging | `develop` | `staging--wholesale-deal-hub.web.app` | Testing before production |
| Production | `main` | `wholesale-deal-hub.web.app` | Live for real users |

## GitHub Actions Pipeline

### On Pull Request (PR) → `develop`

```yaml
# .github/workflows/pr-check.yml
name: PR Check

on:
  pull_request:
    branches: [develop, main]

jobs:
  check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci

      - name: Type check
        run: npx nuxi typecheck

      - name: Lint
        run: npm run lint

      - name: Build
        run: npm run build

      - name: Test (if tests exist)
        run: npm test --if-present
```

### On Push to `develop` → Deploy to Staging

```yaml
# .github/workflows/deploy-staging.yml
name: Deploy Staging

on:
  push:
    branches: [develop]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run build

      - name: Deploy to Firebase Hosting (Staging)
        uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT }}
          channelId: staging
          projectId: wholesale-deal-hub
```

### On Push to `main` → Deploy to Production

```yaml
# .github/workflows/deploy-production.yml
name: Deploy Production

on:
  push:
    branches: [main]

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      - run: npm ci
      - run: npm run build

      - name: Deploy to Firebase Hosting (Production)
        uses: FirebaseExtended/action-hosting-deploy@v0
        with:
          repoToken: ${{ secrets.GITHUB_TOKEN }}
          firebaseServiceAccount: ${{ secrets.FIREBASE_SERVICE_ACCOUNT }}
          channelId: live
          projectId: wholesale-deal-hub

      - name: Deploy Cloud Functions
        run: |
          cd functions
          npm ci
          npx firebase-tools deploy --only functions --token ${{ secrets.FIREBASE_TOKEN }}
```

## Setup Steps

### 1. Initialize Git Repository

```bash
git init
git add .
git commit -m "Initial commit"
```

### 2. Create GitHub Repository

```bash
gh repo create wholesale-deal-hub --private
git remote add origin https://github.com/your-username/wholesale-deal-hub.git
git push -u origin main
```

### 3. Set Up Secrets

In GitHub → Repository → Settings → Secrets → Actions:

- `FIREBASE_SERVICE_ACCOUNT`: JSON key from Firebase (for deployment)
- `FIREBASE_TOKEN`: From `firebase login:ci` (for Cloud Functions)

### 4. Branch Strategy

```
main ← production deployments
  ↑
develop ← staging deployments, integration
  ↑
feature/phase-X-feature ← individual work
```

Workflow:
1. Create feature branch from `develop`
2. Work on feature
3. Create PR to `develop` → CI checks run
4. Merge to `develop` → auto-deploy to staging
5. Test on staging
6. Create PR from `develop` to `main` → CI checks run
7. Merge to `main` → auto-deploy to production

### 5. Firebase Hosting Configuration

**File**: `firebase.json`

```json
{
  "hosting": {
    "public": ".output/public",
    "ignore": ["firebase.json", "**/.*", "**/node_modules/**"],
    "rewrites": [
      { "source": "**", "destination": "/index.html" }
    ],
    "headers": [
      {
        "source": "**/*.@(js|css)",
        "headers": [
          { "key": "Cache-Control", "value": "public, max-age=31536000, immutable" }
        ]
      }
    ]
  },
  "firestore": {
    "rules": "firestore.rules",
    "indexes": "firestore.indexes.json"
  },
  "storage": {
    "rules": "storage.rules"
  }
}
```

### 6. Error Monitoring (Optional)

Add error tracking for production:

**Option A**: Firebase Crashlytics (for web — limited but free)

**Option B**: Sentry (free tier: 5K errors/month)

```bash
npm install @sentry/vue
```

```typescript
// plugins/sentry.client.ts
import * as Sentry from '@sentry/vue'

export default defineNuxtPlugin((nuxtApp) => {
  if (!import.meta.dev) {
    Sentry.init({
      app: nuxtApp.vueApp,
      dsn: 'your-sentry-dsn',
      tracesSampleRate: 0.1,
    })
  }
})
```

## Domain Setup (Optional)

2. Add DNS records at your domain registrar
3. Firebase provisions SSL automatically
