# 9.3 — Cost Management

## What We're Building

Understanding and monitoring Firebase costs so the app runs free as long as possible, with a clear plan for when to scale.

## Firebase Free Tier (Spark Plan)

| Service | Daily Limit | Monthly Equivalent |
|---------|------------|-------------------|
| Firestore reads | 50,000/day | ~1.5M/month |
| Firestore writes | 20,000/day | ~600K/month |
| Firestore deletes | 20,000/day | ~600K/month |
| Firebase Auth (phone) | 10,000 verifications/month | 10K/month |
| Storage | 5GB total | 5GB |

## Usage Projections

### Small Scale (Launch — 1 Wholesaler)

Assumptions:
- 1 wholesaler, 20 retailers, 3 delivery boys
- 30 orders/day
- 50 products in catalog
- 5 page views per user per day

**Daily reads estimate**:
- 20 retailers × 5 visits × (product fetch + order fetch) = ~400 reads
- Wholesaler dashboard × 10 visits × order list = ~100 reads
- Delivery 3 boys × 10 visits × assigned orders = ~60 reads
- Real-time listeners: ~500 reads/day (depends on how frequently orders change)
- **Total: ~1,060 reads/day** (well within 50K limit)

**Daily writes estimate**:
- 30 orders × 1 write each = 30
- 30 orders × 2 status changes average = 60
- Product updates: ~5/day = 5
- Activity logs: ~100/day = 100
- **Total: ~195 writes/day** (well within 20K limit)

**Verdict**: Free tier handles this easily. You could 10x this and still be fine.

### Medium Scale (Growth — 5 Wholesalers)

Assumptions:
- 5 wholesalers, 100 retailers, 15 delivery boys
- 150 orders/day
- 250 total products
- 10 page views per user per day

**Daily reads**: ~15,000 (still within 50K)
**Daily writes**: ~1,500 (still within 20K)

**Verdict**: Still on free tier, but getting closer to limits.

### When to Upgrade

Upgrade to **Blaze Plan** (pay-as-you-go) when:
- Daily reads consistently exceed 40,000
- Phone verifications exceed 8,000/month
- Storage exceeds 4GB
- You need more Cloud Functions compute

**Blaze Plan pricing**:
- Firestore reads: $0.06 per 100K reads
- Firestore writes: $0.18 per 100K writes
- At medium scale: ~$5-15/month

## Cost Optimization Strategies

### 1. Minimize Firestore Reads

- **Cache aggressively**: Firestore offline persistence avoids repeated reads
- **Fetch once, filter client-side**: Load all products once (1 read per document), filter in memory
- **Use Firestore snapshots**: `onSnapshot` with local changes doesn't count as a read for cached docs
- **Paginate**: Don't load all orders. Load 20, load more on scroll.

### 2. Minimize Firestore Writes

- **Batch writes**: Use `writeBatch` for bulk operations (1 write per document, but 1 network round-trip)
- **Don't log excessively**: Activity logs are writes. Log important actions, not every click.
- **Debounce updates**: If the user rapidly edits an order, debounce the Firestore write.

### 3. Reduce Phone Auth Costs

- Phone verifications are limited to 10K/month on free tier
- Users only verify once (session persists)
- Test phone numbers don't count against the quota
- If users log in daily: 20 users × 30 days = 600 verifications/month (safe)

### 4. Control Storage

- Compress images (done in Phase 4/8)
- Limit images per product (5 max)
- Clean up orphaned images (deleted products)
- Don't store voice recordings (transcribe and discard)

## Monitoring Dashboard

### Firebase Console

Use the built-in Firebase console Usage tab to monitor:
- Daily reads/writes/deletes
- Storage usage
- Auth verifications
- Hosting bandwidth

### Alerts

Set up Firebase billing alerts:
1. Firebase Console → Settings → Billing
2. Set a budget alert at $0 (alerts when you'd start paying on Blaze)
3. Set alerts at $5, $10, $25

### In-App Admin Stats (Optional)

Add a section to the wholesaler dashboard:
- Current month's estimated Firestore reads
- Storage used
- Active users count

This helps the wholesaler understand the system's scale.

## Migration Plan (If Leaving Firebase)

If Firebase becomes too expensive or limiting:

**Option A: Stay Firebase, add caching layer**
- Add Redis/Memcached via Cloud Run
- Cache frequently read data
- Reduces Firestore reads significantly

**Option B: Move to Supabase**
- Open-source Firebase alternative
- PostgreSQL (real SQL queries)
- Row-level security
- More generous free tier
- Migration effort: moderate (rewrite data access layer)

**Option C: Custom backend**
- Express/Fastify + PostgreSQL
- Full control
- Higher maintenance
- Migration effort: significant

**Recommendation**: Stay on Firebase through Phase 9. Only reconsider if costs exceed $50/month or you hit fundamental Firestore limitations.
