# 6.1 — Delivery Interface

## What We're Building

The delivery boy's view of the application. A simple, mobile-first interface showing assigned orders and allowing status updates.

## Pages

### Active Orders

**Page**: `/delivery/orders`

Shows all orders assigned to the current delivery user that are not yet delivered.

Two sections:
1. **To Pick Up**: Orders with status `accepted` (assigned but not picked up yet)
2. **In Transit**: Orders with status `shipped` (picked up, on the way)

Each order card shows:
- Retailer name
- Retailer address
- Retailer phone (tap to call)
- Item count and total value
- Time since assignment
- Action button (context-dependent)

### Delivery History

**Page**: `/delivery/history`

Shows completed deliveries (status `delivered`). Sorted newest first. Useful for:
- Delivery boy tracking their daily work
- Wholesaler reviewing delivery performance

### Order Detail (Delivery View)

**Page**: `/delivery/orders/[id]`

Full order details:
- All items with quantities (read-only — delivery can't edit items)
- Retailer contact info
- Order notes (from retailer)
- Admin notes (from wholesaler, if any relevant to delivery)
- Status actions

## Layout

**File**: `layouts/delivery.vue`

Simple mobile layout:
- Top header with app name and delivery boy's name
- Bottom tab bar: [Active Orders] [History] [Profile]
- No sidebar (mobile-first)
