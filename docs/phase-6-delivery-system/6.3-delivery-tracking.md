# 6.3 — Delivery Tracking

## What We're Building

Real-time visibility into delivery status for all three roles. The retailer knows when their order is being delivered, the wholesaler sees the overall delivery pipeline, and the delivery boy has a clear workflow.

## Status Updates (Delivery → System)

When the delivery boy taps a status button:

### "Mark Picked Up"

1. Updates order status: `accepted` → `shipped`
2. Appends to `statusHistory`:
   ```json
   { "status": "shipped", "changedBy": "deliveryUserId", "changedAt": "timestamp" }
   ```
3. Sets `updatedAt` to current timestamp
4. Shows confirmation toast: "Order marked as picked up"

### "Mark Delivered"

1. Updates order status: `shipped` → `delivered`
2. Appends to `statusHistory`
3. Sets `updatedAt`
4. Optional: capture delivery proof
   - Photo of delivered items (uses camera)
   - Stored in Firebase Storage at `deliveries/{orderId}/proof.jpg`
   - URL saved in order document as `deliveryProofUrl`
5. Shows confirmation toast: "Order delivered!"

### Delivery Confirmation (Optional Enhancement)

Before marking as delivered, the delivery boy can:
- Take a photo (proof of delivery)
- Add a note ("Left with store assistant, owner was out")

This is optional for MVP but valuable for dispute resolution.

## Real-Time Updates

Use Firestore `onSnapshot` for live delivery tracking:

**Retailer**: Subscribe to their own order document. When status changes, the timeline updates automatically. No page refresh needed.

**Wholesaler**: Subscribe to orders with `status in ['accepted', 'shipped']`. Dashboard counts update live.

**Delivery boy**: Subscribe to their assigned orders. When a new order is assigned, it appears instantly.

```typescript
function subscribeToOrder(orderId: string, callback: (order: Order) => void) {
  return onSnapshot(doc(db, 'orders', orderId), (snapshot) => {
    if (snapshot.exists()) {
      callback({ id: snapshot.id, ...snapshot.data() } as Order)
    }
  })
}
```

Remember to unsubscribe when the component is unmounted:

```typescript
onUnmounted(() => {
  if (unsubscribe) unsubscribe()
})
```

## Notifications (Prep for Phase 7)

When delivery status changes, the affected parties should be notified:
- Retailer: "Your order has been picked up / delivered"
- Wholesaler: "Suresh delivered order #abc123"

For now (pre-PWA), these show as in-app notifications (if the user has the app open). Push notifications come in Phase 7.

### In-App Notification Store

**File**: `stores/notifications.ts`

```typescript
interface Notification {
  id: string
  type: 'order_update' | 'delivery_update' | 'system'
  title: string
  message: string
  read: boolean
  createdAt: Date
  link?: string  // route to navigate to
}
```

Show a bell icon with unread count in the header. Clicking opens a notification dropdown.
