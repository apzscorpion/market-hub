# 6.2 — Order Assignment

## What We're Building

The wholesaler's ability to assign orders to delivery personnel. After accepting an order, the wholesaler selects which delivery boy should fulfill it.

## Assignment Flow

```
1. Wholesaler views an accepted order
2. Clicks "Assign Delivery"
3. Dropdown shows available delivery boys
4. Selects one
5. Order updates:
   - assignedDeliveryId = selected delivery boy's ID
   - assignedDeliveryName = delivery boy's name (denormalized)
   - status remains "accepted" (it changes to "shipped" when delivery picks up)
6. Delivery boy sees the order appear in their Active Orders
```

### Delivery Boy Selector

Shows all delivery users with:
- Name
- Current active order count (helps balance workload)
- Online/active status indicator
- Blocked users shown but not selectable

### Workload Display

To help the wholesaler balance deliveries:
- Show how many active (undelivered) orders each delivery boy has
- Sort by fewest active orders first
- Optionally show the last delivery location/area

## Assignment from Order List

In addition to the detail page, allow quick assignment from the order list:
- Each order card shows "Assign" button (if no delivery assigned)

- Clicking opens a small popover with the delivery selector

## Edge Cases

| Scenario | Handling |
|----------|---------|
| No delivery boys exist | Show "Add delivery personnel in User Management" |
| All delivery boys are blocked | Show "No active delivery personnel available" |
| Delivery boy is blocked after assignment | Order stays assigned but show warning |
| Order already shipped | Can't reassign (delivery already picked up) |
| Delivery boy has too many orders | Show warning but allow assignment |
