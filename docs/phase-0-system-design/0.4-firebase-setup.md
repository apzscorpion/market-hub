# 0.4 — Firebase Setup

## What We're Building

A fully configured Firebase project with Authentication, Firestore, and Storage enabled. Plus initial security rules that protect the data from day one.

---

## Step 1: Create Firebase Project

1. Go to [Firebase Console](https://console.firebase.google.com)
2. Click "Add Project"
3. Name: `wholesale-deal-hub`
4. Disable Google Analytics (not needed now, can enable later)
5. Click Create

## Step 2: Register Web App

1. In Firebase Console → Project Settings → Your Apps
2. Click the web icon (`</>`)
3. App nickname: `wholesale-deal-hub-web`
4. Check "Also set up Firebase Hosting"
5. Copy the config object — you'll need it

The config looks like:

```typescript
const firebaseConfig = {
  apiKey: "...",
  authDomain: "wholesale-deal-hub.firebaseapp.com",
  projectId: "wholesale-deal-hub",
  storageBucket: "wholesale-deal-hub.appspot.com",
  messagingSenderId: "...",
  appId: "..."
}
```

Store this in an `.env` file (never commit to git):

```
NUXT_PUBLIC_FIREBASE_API_KEY=...
NUXT_PUBLIC_FIREBASE_AUTH_DOMAIN=...
NUXT_PUBLIC_FIREBASE_PROJECT_ID=...
NUXT_PUBLIC_FIREBASE_STORAGE_BUCKET=...
NUXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NUXT_PUBLIC_FIREBASE_APP_ID=...
```

## Step 7: Firestore Security Rules

Create `firestore.rules`:

```
rules_version = '2';

service cloud.firestore {
  match /databases/{database}/documents {

    // Helper: check if the requester is authenticated
    function isSignedIn() {
      return request.auth != null;
    }

    // Helper: get user's role from their user document
    function getUserRole() {
      return get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role;
    }

    function isWholesaler() {
      return isSignedIn() && getUserRole() == 'wholesaler';
    }

    function isRetailer() {
      return isSignedIn() && getUserRole() == 'retailer';
    }

    function isDelivery() {
      return isSignedIn() && getUserRole() == 'delivery';
    }

    // Users collection
    match /users/{userId} {
      // Users can read their own document
      allow read: if isSignedIn() && request.auth.uid == userId;
      // Wholesaler can read/write any user
      allow read, write: if isWholesaler();
    }

    // Products collection
    match /products/{productId} {
      // Anyone signed in can read products
      allow read: if isSignedIn();
      // Only wholesaler can create/update/delete products
      allow write: if isWholesaler();
    }

    // Categories collection
    match /categories/{categoryId} {
      allow read: if isSignedIn();
      allow write: if isWholesaler();
    }

    // Orders collection
    match /orders/{orderId} {
      // Retailers can read their own orders
      allow read: if isSignedIn() &&
        (resource.data.retailerId == request.auth.uid ||
         isWholesaler() ||
         resource.data.assignedDeliveryId == request.auth.uid);

      // Retailers can create orders (for themselves only)
      allow create: if isRetailer() &&
        request.resource.data.retailerId == request.auth.uid;

      // Retailers can update their own pending orders
      allow update: if isRetailer() &&
        resource.data.retailerId == request.auth.uid &&
        resource.data.status == 'pending';

      // Wholesaler can update any order
      allow update: if isWholesaler();

      // Delivery can update status on their assigned orders
      allow update: if isDelivery() &&
        resource.data.assignedDeliveryId == request.auth.uid;
    }

    // Activity logs
    match /activityLogs/{logId} {
      allow read: if isWholesaler();
      allow create: if isSignedIn();
    }
  }
}
```

**Important**: The `getUserRole()` function reads a Firestore document on every request. This costs 1 read per security rule evaluation. At scale, consider moving roles to Firebase Auth custom claims. For MVP, this is fine.

## Step 8: Storage Security Rules

Create `storage.rules`:

```
rules_version = '2';

service firebase.storage {
  match /b/{bucket}/o {
    // Product images — anyone signed in can read, only wholesaler can write
    match /products/{allPaths=**} {
      allow read: if request.auth != null;
      allow write: if request.auth != null; // tighten later with role check
    }
  }
}
```

## Step 9: Firebase Emulator Setup

The emulator lets you develop without touching production data.

After `firebase init`, your `firebase.json` should include:

```json
{
  "emulators": {
    "auth": { "port": 9099 },
    "firestore": { "port": 8080 },
    "storage": { "port": 9199 },
    "ui": { "enabled": true, "port": 4000 }
  }
}
```

Run with: `firebase emulators:start`

In your Nuxt app, connect to emulators in dev mode:

```typescript
if (process.dev) {
  connectAuthEmulator(auth, 'http://localhost:9099')
  connectFirestoreEmulator(db, 'localhost', 8080)
  connectStorageEmulator(storage, 'localhost', 9199)
}
```
