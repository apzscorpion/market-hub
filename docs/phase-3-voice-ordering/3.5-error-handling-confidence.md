# 3.5 — Error Handling & Confidence

## What We're Building

Robust handling for when voice recognition fails, produces garbage, or matches the wrong products. The system should degrade gracefully and never leave the user stuck.

## Error Types and Handling

### 1. Microphone Access Denied

**Cause**: User denied mic permission, or device has no mic.

**Handling**:
- Show persistent banner: "Microphone access is needed for voice ordering"
- Link to browser settings to enable mic
- Hide mic button and show "Voice unavailable" tooltip
- Manual ordering still works perfectly

### 2. Speech Recognition Returns Nothing

**Cause**: Ambient noise, user didn't speak clearly, or API failed.

**Handling**:
- After 10 seconds of silence, stop listening automatically
- Show: "We didn't catch that. Try again in a quieter environment."
- Let user retry immediately

### 3. Transcript is Gibberish

**Cause**: Background noise, thick accent, language mismatch.

**Handling**:
- Parser returns empty items or items with no product match
- Show the raw transcript to the user: "We heard: [gibberish text]"
- Prompt: "This doesn't look right. Try again or add items manually."
- Provide "Clear and retry" button

### 4. Wrong Product Matched

**Cause**: "vanilla" matched "Vanilla Cup" but user meant "Vanilla Big Pack".

**Handling**:
- Medium/low confidence indicators (from 3.4) alert the user
- Clicking the confidence indicator shows alternative matches
- User selects the correct one from alternatives dropdown
- The table row updates

### 5. Partial Recognition

**Cause**: User said 5 items but only 3 were recognized.

**Handling**:
- Show what was recognized
- Show the full transcript below the table
- Let user manually add the missing items
- Don't auto-submit — always require explicit "Place Order" tap

### 6. Network Error (Web Speech API)

**Cause**: Web Speech API requires internet (it sends audio to Google's servers).

**Handling**:
- Detect offline: `!navigator.onLine`
- Show: "Voice ordering requires an internet connection"
- Suggest: "Add items manually while offline"

## Confidence UI System

### Confidence Indicator Component

Each row in the order table gets a small indicator:

| Confidence | Icon | Color | Meaning |
|-----------|------|-------|---------|
| High | ✅ | Green | Confident match, no action needed |
| Medium | ⚠️ | Yellow | Likely correct but verify |
| Low | ❓ | Orange | Multiple matches, user should pick |
| None | ❌ | Red | No match found, manual entry needed |
| Manual | (none) | (none) | User typed this manually, no indicator |

The indicator disappears once the user confirms or edits the row.

### Confidence Banner

If any rows have low/no confidence, show a banner above the table:

```
┌──────────────────────────────────────────────┐
│ ⚠️ 2 items need your attention               │
│ Review the highlighted rows before ordering   │
└──────────────────────────────────────────────┘
```

### Prevent Submission with Unresolved Items

The "Place Order" button should be disabled (or show warning) if:
- Any row has `confidence: 'none'` (no product matched)
- Any row has empty product field

For `medium` and `low` confidence: allow submission but show a confirmation dialog: "Some items were matched by voice. Are you sure these are correct?"

## Voice Session Logging

For improving the system over time, log voice sessions:

```typescript
interface VoiceSession {
  id: string
  userId: string
  language: string
  rawTranscript: string
  parsedItems: ParsedItem[]
  matchedItems: MatchedItem[]
  userCorrections: { original: string, corrected: string }[]
  timestamp: Date
}
```

Store in a `voiceLogs` Firestore collection (wholesaler can review in Phase 5).

This data is gold for:
- Finding products that need more aliases
- Understanding common recognition failures
- Measuring voice accuracy over time

## Retry Flow

```
Voice fails → Show transcript + error
  ├── [Retry] → Clear transcript, restart mic
  ├── [Edit manually] → Focus on table, dismiss voice UI
  └── [Use as text] → Treat transcript as-is, parse whatever matches
```
