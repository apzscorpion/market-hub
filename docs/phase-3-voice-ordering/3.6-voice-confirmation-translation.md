# 3.6 — Voice Confirmation & Translation Layer

## What We're Building

Two features that complete the voice experience:
1. **Voice Confirmation** — The user says "Confirm" to place an order without tapping any button
2. **Translation Layer** — Explicit Malayalam → English mapping as a distinct step in the voice pipeline

## Part 1: Voice Confirmation

### The Dream UX

```
Retailer: "5 vanilla, 2 choco bar, 10 strawberry cone"
  → Table fills automatically
Retailer reviews table...
Retailer: "Confirm"
  → Order placed ✅
```

Zero taps. Pure voice. This is the endgame for power users.

### How It Works

After voice input populates the table, keep the mic listening (or allow re-activation):

1. Table is populated with voice items
2. User reviews (edits if needed)
3. User taps mic again (or mic stays active in "command mode")
4. User says a confirmation phrase
5. System recognizes the command → triggers order placement

### Confirmation Keywords

Recognize multiple confirmation phrases in both languages:

```typescript
const confirmationPhrases = {
  en: ['confirm', 'confirmed', 'place order', 'submit', 'done', 'okay order', 'send'],
  ml: ['ഉറപ്പാക്കുക', 'ഓർഡർ ചെയ്യുക', 'അയക്കുക', 'ശരി', 'ഓക്കേ'],
}
```

### Safety: Double Confirmation

Voice confirmation MUST have a safety net:

```
User says "Confirm"
  → System shows confirmation modal:
    "Place order? 5 items, ₹1,850"
    [Confirm ✅]  [Cancel ❌]
  → User taps Confirm OR says "Yes" / "ശരി"
    → Order placed
```

Why: Voice recognition errors could accidentally place an order. The modal acts as a "did you really mean it?" gate. Tapping OR speaking can dismiss it.

### Cancel Keywords

Also support voice cancellation:

```typescript
const cancellationPhrases = {
  en: ['cancel', 'no', 'stop', 'go back', 'clear'],
  ml: ['വേണ്ട', 'നിർത്തുക', 'മാറ്റുക'],
}
```

### Command vs Order Mode

The mic operates in two modes:

1. **Order Mode** (default): Speech is parsed as product-quantity pairs
2. **Command Mode**: Speech is checked against commands (confirm, cancel, clear, etc.)

How to switch:
- After table is populated, mic switches to Command Mode
- In Command Mode, if speech doesn't match any command, switch back to Order Mode and append items
- Visual indicator shows which mode is active

```
Order Mode:   🎤 "Speaking order..."
Command Mode: 🎤 "Waiting for command..."
```

## Part 2: Translation Layer

### Why a Separate Layer

The voice pipeline from Phase 3.1-3.5 handles Malayalam input via aliases — Malayalam words match directly to products. But there's a gap:

When the speech API returns Malayalam text, the parser needs to:
1. Recognize Malayalam number words (done in 3.3)
2. Match Malayalam product names to aliases (done in 3.4)
3. BUT: some Malayalam phrases need **translation before parsing**

Example:
- "ഐസ്ക്രീം" → "ice cream" (generic term, not a product name)
- "വലിയ പാക്ക്" → "big pack" (size descriptor)
- "ചെറിയ" → "small" (size descriptor)

Without translation, the parser sees "ഐസ്ക്രീം വലിയ പാക്ക്" as a single product query. With translation, it becomes "ice cream big pack" which matches better.

### Translation Dictionary

**File**: `utils/translationDict.ts`

A static dictionary of Malayalam → English mappings for product-related terms:

```typescript
const mlToEn: Record<string, string> = {
  // Product types
  'ഐസ്ക്രീം': 'ice cream',
  'കോൺ': 'cone',
  'ബാർ': 'bar',
  'കപ്പ്': 'cup',
  'സ്റ്റിക്ക്': 'stick',

  // Sizes
  'വലിയ': 'big',
  'ചെറിയ': 'small',
  'പാക്ക്': 'pack',
  'ഫാമിലി': 'family',

  // Flavors
  'വാനില': 'vanilla',
  'ചോക്ലേറ്റ്': 'chocolate',
  'സ്ട്രോബെറി': 'strawberry',
  'മാങ്ങ': 'mango',
  'പിസ്ത': 'pista',

  // Common terms
  'എണ്ണം': 'pieces',
  'ബോക്സ്': 'box',
  'കാർട്ടൺ': 'carton',
}
```

### Where Translation Fits in the Pipeline

Updated pipeline:

```
Layer 1: Speech Recognition → Raw text
Layer 2: Translation & Normalization
  a. Remove filler words (both languages)
  b. Translate known Malayalam terms to English
  c. Normalize Unicode
  d. Result: Clean, mostly-English text
Layer 3: Alias Matching → Match to products
Layer 4: Order Parser → Extract qty-product pairs
Layer 5: Voice Commands → Detect confirm/cancel/etc.
```

### Translation Function

```typescript
function translateToEnglish(text: string): string {
  let result = text

  // Sort by length (longest first) to avoid partial replacements
  const entries = Object.entries(mlToEn)
    .sort((a, b) => b[0].length - a[0].length)

  for (const [ml, en] of entries) {
    result = result.replace(new RegExp(ml, 'g'), en)
  }

  return result.trim()
}
```

**Important**: This is NOT machine translation (no Google Translate API). It's a curated dictionary of product-related terms. It's fast (pure string replacement), free, and works offline.

### When NOT to Translate

If the input is already mostly English (detected by character analysis), skip translation. Malayalam characters are in the Unicode range U+0D00-U+0D7F:

```typescript
function isMalayalamHeavy(text: string): boolean {
  const malayalamChars = text.match(/[\u0D00-\u0D7F]/g)?.length ?? 0
  return malayalamChars > text.length * 0.3 // >30% Malayalam
}
```
