# 3.1 — Speech-to-Text

## What We're Building

The audio capture and transcription layer. The user speaks, and we get text. This sub-phase focuses on getting raw text — parsing it into structured data comes in 3.2.

## Approach: Start with Web Speech API

The browser's built-in Web Speech API is the right starting point:
- **Free** — no API costs
- **No server needed** — runs entirely in the browser
- **Real-time** — transcription happens as the user speaks
- **Malayalam support** — available on Chrome (uses Google's servers internally)

### Web Speech API Basics

```typescript
const recognition = new webkitSpeechRecognition()
recognition.continuous = true      // keep listening (don't stop after one phrase)
recognition.interimResults = true  // show partial results while speaking
recognition.lang = 'ml-IN'        // Malayalam (India) or 'en-IN' for English

recognition.onresult = (event) => {
  const transcript = event.results[event.results.length - 1][0].transcript
  const isFinal = event.results[event.results.length - 1].isFinal
  // Use transcript...
}

recognition.start()
```

### Language Options

| Language Code | Language | When to Use |
|--------------|---------|------------|
| `en-IN` | English (India) | Default for English-speaking retailers |
| `ml-IN` | Malayalam | For Malayalam-speaking retailers |

Allow the user to pick their preferred language. Or better: detect automatically by trying both and seeing which gives higher confidence (advanced — do this later).

For MVP: let the user toggle between English and Malayalam before tapping the mic.

### 2. Build Mic Button Component

**File**: `components/retailer/MicButton.vue`

States:
- **Idle**: Mic icon, gray. Tap to start.
- **Listening**: Mic icon, red/pulsing animation. Tap to stop.
- **Processing**: Spinner. Text is being finalized.
- **Error**: Red icon with error message.

Visual feedback is critical — the user must know the mic is active.

### 3. Build Transcript Display

While the user speaks, show the real-time transcript below the mic button:

```
┌──────────────────────────────────┐
│  🎤 Listening...                  │
│                                  │
│  "5 vanilla 2 choco bar 10..."  │  ← live transcript
│                                  │
│  [Stop & Process]               │
└──────────────────────────────────┘
```

- Gray text for interim results (not yet confirmed)
- Black text for final results
- Scrollable if the text is long

### 4. Handle Browser Compatibility

| Browser | Web Speech API | Notes |
|---------|---------------|-------|
| Chrome (desktop) | Full support | Best accuracy |
| Chrome (Android) | Full support | Good accuracy |
| Safari (iOS) | Partial | May require user gesture |
| Firefox | Not supported | Show "Use Chrome" message |
| Edge | Full support (uses same engine as Chrome) | Works |

Check support on page load:

```typescript
const isSupported = 'webkitSpeechRecognition' in window || 'SpeechRecognition' in window
```

If not supported, hide the mic button and show a tooltip explaining it.

### 5. Handle Permissions

The browser will ask for microphone permission. Handle:
- **Granted**: Proceed normally
- **Denied**: Show message "Microphone access is needed for voice ordering"
- **Dismissed**: Prompt again on next tap

## Upgrade Path: Google Cloud Speech-to-Text

When to upgrade from Web Speech API:
- Need better Malayalam accuracy
- Need offline support
- Need to process pre-recorded audio
- Need custom vocabulary (product-specific language model)

Google Cloud STT setup (Phase 3+):
1. Enable Speech-to-Text API in Google Cloud Console
2. Create a Cloud Function that receives audio and returns text
3. Client records audio using MediaRecorder API
4. Sends audio blob to Cloud Function
5. Cloud Function calls STT API and returns transcript

Cost: ~$0.006 per 15 seconds of audio. For a typical order (30 seconds), that's about ₹1 per order.
