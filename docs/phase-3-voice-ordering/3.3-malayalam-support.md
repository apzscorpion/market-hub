# 3.3 — Malayalam Support

## What We're Building

Extend the voice parser to handle Malayalam input. Retailers in Kerala will often speak in Malayalam, and the system needs to understand product names and numbers in the language.

## Mixed Language Handling

Retailers often mix languages: "5 വാനില and 2 choco bar". The parser needs to handle:

1. **Malayalam numbers + Malayalam product names**: "അഞ്ച് വാനില" (5 vanilla)
2. **English numbers + Malayalam product names**: "5 വാനില"
3. **Malayalam numbers + English product names**: "അഞ്ച് vanilla"
4. **Fully mixed**: "അഞ്ച് vanilla രണ്ട് choco bar"

The parser should:
- Check both English AND Malayalam number word dictionaries
- Not care about the language of the product text (matching happens against aliases which include both languages)

### Updated `parseNumber` Function

```typescript
function parseNumber(token: string): number | null {
  // Try plain digit
  const num = parseInt(token, 10)
  if (!isNaN(num)) return num

  // Try English word
  const eng = englishNumbers[token.toLowerCase()]
  if (eng !== undefined) return eng

  // Try Malayalam word
  const ml = malayalamNumbers[token]
  if (ml !== undefined) return ml

  return null
}
```

## Speech-to-Text Language Setting

When using Web Speech API with `lang: 'ml-IN'`, the transcriber:
- Outputs Malayalam script for Malayalam words
- May output Arabic numerals (1, 2, 3) or Malayalam words for numbers
- May mix in English words if the speaker uses them

When using `lang: 'en-IN'`:
- Outputs English text
- Malayalam words may be transliterated to English or dropped

**Recommendation**: Default to `ml-IN` for Malayalam-speaking users. The parser handles mixed text either way.

## Unicode Handling

Malayalam text must be normalized before comparison:

```typescript
function normalizeText(text: string): string {
  return text
    .normalize('NFC')  // normalize Unicode
    .trim()
    .replace(/\s+/g, ' ')  // collapse whitespace
}
```

Malayalam-specific considerations:
- Chillu letters (ൽ, ൾ, ൻ, ൺ, ൿ) have multiple Unicode representations
- Virama (്) and ZWJ (‍) combinations can produce different visual forms
- Always normalize before storing and comparing
