# 3.2 — Voice Parser

## What We're Building

The engine that takes raw transcribed text and extracts structured product-quantity pairs. This is the hardest part of the voice system.

## The Problem

Input (what the user said, transcribed):
```
"5 vanilla 2 choco bar 10 strawberry cone"
```

Expected output:
```json
[
  { "text": "vanilla", "quantity": 5 },
  { "text": "choco bar", "quantity": 2 },
  { "text": "strawberry cone", "quantity": 10 }
]
```

This looks simple, but human speech is messy:
- "Give me 5 vanilla and 2 choco" (filler words)
- "Vanilla 5, choco bar 2" (quantity after product)
- "5 vanilla 2 choco 10 strawberry" (no separators)
- "5 large vanilla" (adjective before product)
- "vanilla big pack 5" (full name with quantity at end)

## Parser Strategy

### Pattern-Based Parser (MVP)

Don't use AI/NLP for this. A rule-based parser handles the common patterns and is predictable.

**Step 1: Clean the text**
- Lowercase
- Remove filler words: "give me", "I need", "please", "and", "also"
- Normalize whitespace
- Replace word-numbers with digits: "five" → 5

**Step 2: Extract number-product pairs**

Scan the text for patterns. The two main patterns:

```
Pattern A: NUMBER + PRODUCT_NAME
  "5 vanilla" → { qty: 5, text: "vanilla" }

Pattern B: PRODUCT_NAME + NUMBER
  "vanilla 5" → { qty: 5, text: "vanilla" }
```

**Step 3: Split into segments**

Between each number-product pair, there's usually a comma, "and", or just a number indicating the start of the next pair.

Algorithm:
1. Tokenize the text into words
2. Scan left to right
3. When a number is found, look at the next words until the next number is found
4. Everything between this number and the next is the product name
5. If no number precedes text, assume quantity = 1

### Implementation

```typescript
interface ParsedItem {
  text: string      // the product name as spoken
  quantity: number   // extracted quantity
  raw: string        // original text segment
}

function parseVoiceText(text: string): ParsedItem[] {
  const cleaned = cleanText(text)
  const tokens = cleaned.split(/\s+/)
  const items: ParsedItem[] = []

  let i = 0
  while (i < tokens.length) {
    const num = parseNumber(tokens[i])

    if (num !== null) {
      // Found a number — collect following words as product name
      const nameTokens: string[] = []
      i++
      while (i < tokens.length && parseNumber(tokens[i]) === null) {
        nameTokens.push(tokens[i])
        i++
      }
      if (nameTokens.length > 0) {
        items.push({
          text: nameTokens.join(' '),
          quantity: num,
          raw: `${num} ${nameTokens.join(' ')}`,
        })
      }
    } else {
      // No number — could be product name with quantity after
      const nameTokens: string[] = []
      while (i < tokens.length && parseNumber(tokens[i]) === null) {
        nameTokens.push(tokens[i])
        i++
      }
      const qty = i < tokens.length ? parseNumber(tokens[i]) ?? 1 : 1
      if (i < tokens.length && parseNumber(tokens[i]) !== null) i++

      if (nameTokens.length > 0) {
        items.push({
          text: nameTokens.join(' '),
          quantity: qty,
          raw: nameTokens.join(' ') + (qty > 1 ? ` ${qty}` : ''),
        })
      }
    }
  }

  return items
}
```

### Number Word Conversion

```typescript
const numberWords: Record<string, number> = {
  'one': 1, 'two': 2, 'three': 3, 'four': 4, 'five': 5,
  'six': 6, 'seven': 7, 'eight': 8, 'nine': 9, 'ten': 10,
  'eleven': 11, 'twelve': 12, 'fifteen': 15, 'twenty': 20,
  'twenty-five': 25, 'thirty': 30, 'fifty': 50, 'hundred': 100,
}

function parseNumber(token: string): number | null {
  const num = parseInt(token, 10)
  if (!isNaN(num)) return num
  return numberWords[token.toLowerCase()] ?? null
}
```

## Limitation: Multi-Word Product Names

"choco bar" is two words. The parser needs to group them together. The current algorithm does this by collecting all non-number words between numbers. But edge cases exist:

- "5 vanilla big pack 2 choco" — is it "vanilla big pack" + "choco"? Or "vanilla big" + "pack 2 choco"?

Solution: After parsing, match each text segment against the product database. "vanilla big pack" matches a product, so keep it as one item. If "vanilla big pack 2 choco" doesn't match, try splitting at different points.

This matching happens in sub-phase 3.4.
