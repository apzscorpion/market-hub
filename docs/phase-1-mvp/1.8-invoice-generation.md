# 1.8 — Invoice Generation

## What We're Building

Automatic invoice creation when an order is confirmed. Every order gets an invoice — no manual work. The invoice is a financial document that the retailer can view, download, and use for their records.

## Why in MVP

Invoices are not a "nice to have" — they're a business necessity. Retailers need invoices for their accounting. Wholesalers need them for tax compliance (GST). Building it early means every order from day one has a proper invoice.

## Invoice Generation Flow

```
1. Retailer places order → status: pending
2. Wholesaler accepts order → status: accepted
3. System auto-generates invoice:
   a. Read order items (snapshot them — prices frozen)
   b. Calculate subtotal, tax, total
   c. Generate sequential invoice number (INV-2026-0001)
   d. Write invoice document to Firestore
   e. Update order with invoiceId reference
4. Invoice is viewable by retailer and wholesaler
```

### 2. Sequential Invoice Numbering

Invoice numbers must be sequential and unique. Use a Firestore transaction:

```typescript
async function getNextInvoiceNumber(): Promise<string> {
  const configRef = doc(db, 'systemSettings', 'config')

  return runTransaction(db, async (transaction) => {
    const configDoc = await transaction.get(configRef)
    const current = configDoc.data()?.nextInvoiceNumber ?? 1
    const prefix = configDoc.data()?.invoicePrefix ?? 'INV'
    const year = new Date().getFullYear()

    transaction.update(configRef, { nextInvoiceNumber: current + 1 })

    return `${prefix}-${year}-${String(current).padStart(4, '0')}`
  })
}
```

This ensures two simultaneous order confirmations don't get the same invoice number.

## Edge Cases

| Scenario | Handling |
|----------|---------|
| Order accepted but invoice fails | Retry. If persistent, log error, notify wholesaler. Order status should still change — invoice can be regenerated. |
| Order cancelled after invoice | Invoice status stays "generated". Add a note: "Order cancelled". Don't delete the invoice (it's a financial record). |
| Price changed between order and accept | Invoice uses prices from the ORDER (frozen at order time), not current product prices. |
| Tax rate changes | New invoices use the new rate. Existing invoices are immutable. |
| GST not configured | Generate invoice with 0% tax. Show warning to wholesaler: "Configure your GST details in Settings". |
