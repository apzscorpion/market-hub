### Dashboard Stats (Top)

Quick numbers at the top:
- Total orders today
- Pending orders (needs action)
- Orders in transit
- Delivered today

### Order List

Shows all orders, newest first. Each order card shows:
- Order ID
- Retailer name + phone
- Item count + total amount
- Status badge (colored)
- Time placed
- "View Details" link

### Filters

- **Status filter**: All / Pending / Accepted / Shipped / Delivered / Cancelled
- **Date filter**: Today / This Week / This Month / All Time
- Filters are URL query params (so you can bookmark filtered views)

## Real-Time Updates (Optional for MVP)

Firestore supports real-time listeners via `onSnapshot`. Instead of fetching orders once, you can subscribe and get live updates when new orders come in.

**MVP**: One-time fetch with manual refresh button.
**Phase 2+**: Switch to real-time listeners.

Why not start with real-time: It adds complexity (managing unsubscribe, handling large datasets), and for MVP, the wholesaler can just refresh.
