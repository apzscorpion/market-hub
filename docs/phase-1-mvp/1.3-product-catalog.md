### Product Card

Each product card shows:
- Product image (placeholder if no image uploaded yet)
- Product name (full name)
- Nickname (smaller, below name)
- Price + unit type (e.g., "вӮ№250 / box")
- "Add to Cart" button
- If already in cart: show quantity controls (вҲ’/+) instead of "Add" button

### Behavior

- Products fetched from Firestore `products` collection where `active == true`
- Ordered by category, then by `sortOrder`
- Search filters by name, nickname, and aliases (client-side filter for MVP, since product count is small)
- Loading skeleton shown while products load
- Empty state if no products exist

### 5. Build Search Bar Component

**File**: `components/common/SearchBar.vue`

Simple text input with debounced search. Emits the search query.

For MVP, search is client-side only (filter the already-loaded product array). No server-side search needed until the product catalog exceeds ~100 items.

## Data Fetch Strategy

**MVP approach**: Fetch all products once on page load. Store in Pinia. Filter/search client-side.

Why this is fine for MVP:
- A typical wholesaler has 20-100 products
- A single Firestore query returns them all
- Client-side filtering is instant
- No need for pagination yet

When to change: If products exceed 500, switch to server-side queries with pagination and Firestore compound queries.
