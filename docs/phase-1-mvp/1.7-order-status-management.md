## What We're Building

The ability for the wholesaler to view order details and change an order's status. This completes the order lifecycle: retailer places → wholesaler reviews → wholesaler progresses status.

## Status Lifecycle

```
pending ──→ accepted ──→ shipped ──→ delivered
   │
   └──→ cancelled
```

Rules:
- **pending → accepted**: Wholesaler reviews and confirms the order
- **accepted → shipped**: Wholesaler hands off to delivery (or ships directly)
- **shipped → delivered**: Delivery boy confirms delivery (Phase 6) OR wholesaler marks it
- **pending → cancelled**: Wholesaler rejects the order
- **No backward transitions**: Can't go from accepted back to pending
- **No skipping**: Can't go from pending directly to shipped

### 2. Build Status Change Logic

**File**: `composables/useOrders.ts` (extend)

Add function: `updateOrderStatus(orderId, newStatus)`

This function:
1. Validates the transition is allowed (pending → accepted, etc.)
2. Updates the `status` field
3. Appends to `statusHistory` array
4. Sets `updatedAt` to `serverTimestamp()`
5. Returns success/failure

```typescript
const allowedTransitions: Record<OrderStatus, OrderStatus[]> = {
  pending: ['accepted', 'cancelled'],
  accepted: ['shipped', 'cancelled'],
  shipped: ['delivered'],
  delivered: [],
  cancelled: [],
}

async function updateOrderStatus(orderId: string, newStatus: OrderStatus) {
  const order = await getOrder(orderId)
  if (!allowedTransitions[order.status].includes(newStatus)) {
    throw new Error(`Cannot transition from ${order.status} to ${newStatus}`)
  }
  // ... Firestore update
}
```

### 3. Build Action Buttons Component

**File**: `components/wholesaler/OrderActions.vue`

Shows different buttons based on current status:

| Current Status | Available Actions |
|---------------|-------------------|
| pending | Accept, Cancel |
| accepted | Mark as Shipped |
| shipped | Mark as Delivered |
| delivered | (none — completed) |
| cancelled | (none — terminal) |

Each button:
- Shows confirmation dialog before executing
- Shows loading state during the Firestore update
- Disables all buttons while one is processing
- Shows success/error toast after completion

### 4. Build Confirmation Dialog

**File**: `components/common/ConfirmDialog.vue`

Modal component:

- Description text explaining the action

For cancellation, include a reason field (optional for MVP, store it if provided).

### 5. Build Status History Component

**File**: `components/common/StatusHistory.vue`

Timeline view showing all status changes:

### 6. Real-Time Status Updates (Retailer Side)

When the wholesaler changes an order's status, the retailer should see it update.

**MVP**: Retailer refreshes the page to see updates.
**Better**: Use Firestore `onSnapshot` on the retailer's order list to get real-time updates.

For MVP, implement the simple approach. But structure the code so switching to real-time is easy.
