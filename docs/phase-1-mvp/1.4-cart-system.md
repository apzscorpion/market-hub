# 1.4 — Cart System

## What We're Building

A simple cart that holds the retailer's selected products and quantities before placing an order. This is a temporary implementation — Phase 2 replaces it with the table-based UI. But for MVP, a standard cart works.

## How the Cart Works

1. Retailer taps "Add to Cart" on a product → product added with qty 1
2. On the product card, qty controls appear (− / +)
3. Retailer can go to Cart page to review all items
4. On the Cart page: edit quantities, remove items, see total
5. "Place Order" button submits the cart as an order

## Cart is Client-Side Only

The cart is NOT stored in Firestore. It lives in Pinia state only.

Why:
- It's temporary data (not yet an order)
- Saves Firestore writes
- Faster interactions
- If the user closes the browser, the cart resets — acceptable for MVP

If persistence is needed later, save to `localStorage` via a Pinia plugin.

## Work Items

### 1. Build Cart Store

**File**: `stores/cart.ts`

```
useCartStore
├── items: CartItem[]          — products in cart
├── itemCount: number          — computed: total items
├── totalAmount: number        — computed: sum of qty * price
├── addItem(product)           — add product or increment if exists
├── removeItem(productId)      — remove from cart
├── updateQuantity(productId, qty) — set specific quantity
├── increment(productId)       — +1
├── decrement(productId)       — -1 (remove if 0)
├── clearCart()                — empty everything
├── getItem(productId)         — find specific item
├── isEmpty: boolean           — computed
```

**CartItem type**:
```typescript
interface CartItem {
  productId: string
  productName: string
  nickname: string
  unitType: string
  price: number
  quantity: number
}
```

│  │      Place Order             │  │

Features:
- List all cart items with name, price, qty controls
- Show line total (qty × price) for each item
- Show subtotal at bottom
- "Place Order" button (disabled if cart empty)
- Remove button (trash icon) per item
- Back button to return to products
- Empty cart state: "Your cart is empty" with link to products

### 3. Build Cart Icon Component

### 4. Build Quantity Controls Component

**File**: `components/common/QuantityControl.vue`

Reusable − / number / + component.

Props:
- `quantity: number`
- `min: number` (default 0)
- `max: number` (default 999)

Emits:
- `update:quantity` (v-model compatible)
- `increment`
- `decrement`

Prevents:
- Going below min
- Non-numeric input

### 5. Update Product Cards

Modify `ProductCard.vue` (from 1.3):
- Check if product is in cart
- If NOT in cart: show "Add" button
- If IN cart: show QuantityControl component

## UX Details

- Adding to cart should give visual feedback (button briefly changes color or shows checkmark)
- Quantity changes are instant (no loading states — it's all local)
- Rupee formatting: use `Intl.NumberFormat('en-IN')` for Indian number format
- If a product is out of stock (future), gray out the Add button
