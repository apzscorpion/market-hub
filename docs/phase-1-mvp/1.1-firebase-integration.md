# 1.1 — Firebase Integration

## What We're Building

The bridge between the Nuxt 3 frontend and Firebase backend. By the end of this sub-phase, the app can talk to Firebase Auth, Firestore, and Storage.

## Work Items

### 1. Create the Firebase Client Plugin

The plugin initializes Firebase when the Nuxt app loads. It only runs on the client side (`.client.ts` suffix).

**File**: `plugins/firebase.client.ts`

What it does:
- Reads Firebase config from runtime config (environment variables)
- Initializes the Firebase app
- Creates Auth, Firestore, and Storage instances
- Connects to emulators in development mode
- Provides all instances via `useNuxtApp().$firebaseAuth`, etc.

### 2. Create Firebase Utility Module

**File**: `utils/firebase.ts`

A standalone module that exports Firebase instances for use in composables and stores that may not have access to the Nuxt plugin context.

### 3. Create `.env` File

**File**: `.env`

Contains all Firebase config values. This file is gitignored.

```
NUXT_PUBLIC_FIREBASE_API_KEY=your-api-key
NUXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
NUXT_PUBLIC_FIREBASE_PROJECT_ID=your-project-id
NUXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project.appspot.com
NUXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=your-sender-id
NUXT_PUBLIC_FIREBASE_APP_ID=your-app-id
```

### 4. Create `.env.example`

Same as `.env` but with placeholder values. This IS committed to git so other developers know what variables are needed.

### 5. Add `.env` to `.gitignore`

Ensure secrets are never committed.

### 6. Test the Connection

Create a temporary test page or use the browser console to verify:
- `firebase.auth()` is accessible
- `firebase.firestore()` can read/write
- Emulators are connected in dev mode

## Key Decisions

**Why client-side only?**
Firebase SDK is designed for the browser. SSR (server-side rendering) with Firebase requires Firebase Admin SDK, which is a different package with different auth. For MVP, client-side only is simpler and sufficient.

**Why both plugin AND util module?**
The plugin provides Firebase via Nuxt's provide/inject. But some code (like Pinia stores) runs outside the component tree and may need direct imports. The util module covers this case.
