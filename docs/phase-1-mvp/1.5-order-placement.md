# 1.5 — Order Placement

## What We're Building

The flow that converts a cart into an order in Firestore. This is where "browsing" becomes "business". The retailer taps "Place Order", the cart items become a Firestore order document, and the wholesaler sees it appear.

## The Order Placement Flow

```
1. Retailer reviews cart → taps "Place Order"
2. Button enters loading state (prevent double-tap)
3. App constructs order document from cart items
4. App writes order to Firestore orders collection
5. On success:
   a. Clear the cart
   b. Show success confirmation (toast + redirect)
   c. Redirect to /retailer/orders (order history)
6. On failure:
   a. Show error message
   b. Cart remains intact (user can retry)
   c. Button re-enables
```

## Critical Error Handling

### Double Submission Prevention
```
1. Set loading = true immediately on button click
2. Disable the button while loading
3. Only re-enable on error
4. On success, navigate away (button no longer exists)
```

### Network Failure
```
1. Firestore write fails → catch error
2. Show error toast: "Failed to place order. Check your connection."
3. Cart remains intact
4. User can retry
```

### Stale Price Protection
When creating the order, use the current product price from the cart (which was loaded from the product catalog). The price is "locked in" at order time — if the wholesaler changes prices later, existing orders aren't affected.
