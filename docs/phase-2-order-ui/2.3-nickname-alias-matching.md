# 2.3 — Nickname & Alias Matching

## What We're Building

The system that lets retailers use short names, Malayalam names, or common misspellings to find products. This is what makes ordering fast — the retailer doesn't need to remember "Vanilla Ice Cream Big Pack 500ml", they just type "vanilla".

## How Nicknames and Aliases Work

Every product has three searchable fields:

| Field | Purpose | Example |
|-------|---------|---------|
| `name` | Full official product name | "Vanilla Ice Cream Big Pack 500ml" |
| `nickname` | Short display name set by wholesaler | "Vanilla Big" |
| `aliases` | Array of alternative names for matching | ["vanilla", "vanila", "വാനില", "വാനിലാ", "van big"] |

### Who Sets These?

- **name**: Set by wholesaler when creating the product
- **nickname**: Set by wholesaler — the short name that appears in the UI
- **aliases**: Set by wholesaler over time. As retailers voice-order and the system encounters new variations, the wholesaler adds them as aliases

### Alias Strategy

Aliases should cover:

1. **Short forms**: "vanilla" for "Vanilla Ice Cream Big Pack"
2. **Malayalam equivalents**: "വാനില" for "Vanilla"
3. **Common misspellings**: "vanila", "vanella"
4. **Voice recognition errors**: Common misheard versions
5. **Abbreviations**: "VBP" for "Vanilla Big Pack"
6. **Informal names**: What retailers actually call it over the phone

### Building the Alias List Over Time

The alias list starts small and grows:

1. **Launch**: Wholesaler adds 2-3 obvious aliases per product
2. **Voice Phase (Phase 3)**: When voice recognition produces unmatched text, log it. Wholesaler reviews and adds as aliases.
3. **Usage data (Phase 5+)**: Track what search queries lead to "No results". These are candidates for new aliases.

## Matching Algorithm

The autocomplete (from 2.2) uses this matching logic:

```typescript
interface MatchResult {
  product: Product
  score: number      // lower = better match
  matchedField: 'nickname' | 'name' | 'alias'
  matchedValue: string
}

function matchProducts(query: string, products: Product[]): MatchResult[] {
  const q = query.toLowerCase().trim()
  const results: MatchResult[] = []

  for (const product of products) {
    // Tier 1: Exact nickname match (score 0)
    if (product.nickname.toLowerCase() === q) {
      results.push({ product, score: 0, matchedField: 'nickname', matchedValue: product.nickname })
      continue
    }

    // Tier 2: Nickname starts with query (score 1)
    if (product.nickname.toLowerCase().startsWith(q)) {
      results.push({ product, score: 1, matchedField: 'nickname', matchedValue: product.nickname })
      continue
    }

    // Tier 3: Alias exact match (score 2)
    const exactAlias = product.aliases.find(a => a.toLowerCase() === q)
    if (exactAlias) {
      results.push({ product, score: 2, matchedField: 'alias', matchedValue: exactAlias })
      continue
    }

    // Tier 4: Alias starts with query (score 3)
    const startsAlias = product.aliases.find(a => a.toLowerCase().startsWith(q))
    if (startsAlias) {
      results.push({ product, score: 3, matchedField: 'alias', matchedValue: startsAlias })
      continue
    }

    // Tier 5: Nickname contains query (score 4)
    if (product.nickname.toLowerCase().includes(q)) {
      results.push({ product, score: 4, matchedField: 'nickname', matchedValue: product.nickname })
      continue
    }

    // Tier 6: Full name contains query (score 5)
    if (product.name.toLowerCase().includes(q)) {
      results.push({ product, score: 5, matchedField: 'name', matchedValue: product.name })
      continue
    }

    // Tier 7: Any alias contains query (score 6)
    const containsAlias = product.aliases.find(a => a.toLowerCase().includes(q))
    if (containsAlias) {
      results.push({ product, score: 6, matchedField: 'alias', matchedValue: containsAlias })
    }
  }

  return results.sort((a, b) => a.score - b.score)
}
```

## Fuzzy Matching (Optional Enhancement)

For even better matching, add fuzzy search using Levenshtein distance. This handles typos:
- "vanila" → "vanilla" (1 character difference)
- "chokolate" → "chocolate" (1 character difference)

Libraries: `fuse.js` (lightweight fuzzy search) or roll your own basic Levenshtein.

**For MVP**: Don't add fuzzy matching. Exact and starts-with matching is sufficient. Add fuzzy in Phase 3 when voice introduces more variation.

## Malayalam-Specific Considerations

Malayalam (മലയാളം) is a Unicode script. Key points:

- Malayalam characters are multi-byte — string length in JS counts correctly with `String.length`
- Searching is case-insensitive (Malayalam doesn't have case, so this is automatic)
- Some Malayalam characters have multiple Unicode representations (e.g., chillu letters). Normalize with `String.normalize('NFC')` before comparing
- Store aliases in normalized form

```typescript
function normalizeText(text: string): string {
  return text.normalize('NFC').toLowerCase().trim()
}
```

The wholesaler needs a UI to manage aliases. This is part of Phase 5 (Admin Panel), but the data structure supports it from the start.
