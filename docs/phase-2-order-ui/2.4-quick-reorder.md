# 2.4 вҖ” Quick Reorder

## What We're Building

A feature that lets retailers repeat a previous order with one tap. Most retailers order similar products every time. Instead of rebuilding the order from scratch, they can load a past order into the table and adjust.

## How It Works

```
1. Retailer opens the "New Order" page
2. They see a "Repeat Previous Order" section above the table
3. Recent orders are shown as cards (last 5)
4. Retailer taps one вҶ’ its items load into the table
5. Retailer edits quantities, adds/removes items as needed
6. Submits as a new order
```

## Work Items

### 1. Fetch Recent Orders

Get the retailer's last 5 orders from Firestore (already available via `useOrders`). Filter to delivered/accepted orders (don't show cancelled ones вҖ” they're probably mistakes).

### 2. Build Reorder Card Component

**File**: `components/retailer/ReorderCard.vue`

Shows:
- Order date
- Item count
- Total amount
- "Load" button

Small, horizontal-scrollable cards. Shouldn't take up too much vertical space.

### 3. Load Order Into Table

When "Load" is tapped:
1. Clear current table rows
2. For each item in the selected order:
   a. Look up the product in the product store (it might have been updated вҖ” new price, etc.)
   b. If product still exists and is active: create a row with current price (NOT the order's historical price)
   c. If product no longer exists: create a row with a warning indicator
3. Add an empty row at the bottom
4. Show a toast: "Loaded order from Apr 10. Review and adjust."

**Use current prices, not historical**: The old order stored price-at-time-of-order. But when reordering, the retailer should see today's prices. This avoids confusion.

**Handle missing products**: If a product was deleted or deactivated since the last order, show the row in red with "Product no longer available" and let the retailer remove it.

### 4. Confirmation Before Overwrite

If the table already has items when the retailer taps "Load", show a confirmation:
- "This will replace your current order. Continue?"
- "Replace" / "Cancel"

Alternative: Merge items instead of replacing. But for simplicity, replace is clearer.

## Edge Cases

| Scenario | Handling |
|----------|---------|
| Retailer has no previous orders | Hide the reorder section entirely |
| Product price changed | Use current price, not historical |
| Product deleted/deactivated | Show warning, let user remove row |
| Same product in old order twice | Merge into single row with combined qty |
| Table already has items | Ask for confirmation before replacing |
