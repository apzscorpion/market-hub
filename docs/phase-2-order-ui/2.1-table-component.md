# 2.1 — Table Component

## What We're Building

The core editable table component that replaces the shopping cart. This is the central UI element of the entire application — retailers will spend 90% of their time interacting with this table.

## Component: `OrderTable.vue`

**File**: `components/retailer/OrderTable.vue`

### Table Structure

| Column | Width | Editable | Behavior |
|--------|-------|----------|----------|
| # | 40px | No | Row number (auto) |
| Product | 40% | Yes | Autocomplete input (covered in 2.2) |
| Qty | 80px | Yes | Number input |
| Unit | 100px | No | Auto-filled from product |
| Price | 100px | No | Auto-filled from product |
| Total | 100px | No | Calculated: qty × price |
| Actions | 40px | No | Delete button |

### Row Data Model

```typescript
interface OrderRow {
  id: string          // unique row ID (for Vue key)
  productId: string   // empty until product selected
  productName: string // display name
  nickname: string
  quantity: number    // default 1
  unitType: string    // auto-filled
  price: number       // auto-filled
  total: number       // computed
}
```

### Behaviors

**Adding rows**:
- An empty row always exists at the bottom of the table
- When the user starts typing in the empty row's product cell, a new empty row appears below
- "Add Row" button also adds a row (for users who prefer clicking)

**Removing rows**:
- Trash icon on each row (except the last empty one)
- Backspace on an empty product cell removes the row and moves focus up
- Minimum 1 row always present

**Editing quantity**:
- Click to select, type new value
- Up/Down arrow keys increment/decrement
- Must be positive integer (validate on input)
- Default is 1 when a product is first selected

**Total calculation**:
- Per row: `quantity × price`
- Grand total: sum of all row totals
- Updates in real-time as user types

### Keyboard Navigation

| Key | Action |
|-----|--------|
| Tab | Move to next cell (product → qty → next row product) |
| Shift+Tab | Move to previous cell |
| Enter | In product cell: select highlighted autocomplete option. In qty cell: move to next row product. |
| Delete/Backspace | In empty product cell: remove row |
| Arrow Up/Down | In qty cell: increment/decrement. In product cell: navigate autocomplete suggestions. |
| Escape | Close autocomplete dropdown |

### Mobile Considerations

On mobile (< 640px):
- Hide Price column (still show Total)
- Product column takes more space
- Each row might render as a card instead of table row
- Tap to edit instead of keyboard focus

## Implementation Approach

Use a `ref<OrderRow[]>` for the table data. Each row is reactive. The table renders with `v-for` over the rows.

Don't use a third-party table library. The requirements are specific enough that a custom component is simpler than configuring a generic library.

```vue
<template>
  <table>
    <thead>...</thead>
    <tbody>
      <tr v-for="(row, index) in rows" :key="row.id">
        <td>{{ index + 1 }}</td>
        <td>
          <ProductAutocomplete v-model="row.productName"
            @select="onProductSelect(row, $event)" />
        </td>
        <td>
          <input type="number" v-model.number="row.quantity"
            @keydown="handleQtyKeydown(index, $event)" />
        </td>
        <td>{{ row.unitType }}</td>
        <td>{{ formatCurrency(row.price) }}</td>
        <td>{{ formatCurrency(row.total) }}</td>
        <td>
          <button @click="removeRow(index)">🗑</button>
        </td>
      </tr>
    </tbody>
  </table>
</template>
```

## Connecting to Order Placement

The table replaces the cart. When "Place Order" is clicked:

1. Filter out empty rows (no product selected)
2. Validate all rows have qty > 0
3. Map rows to `OrderItem[]` format
4. Call `placeOrder()` from the order composable (same as Phase 1)
5. Clear the table on success

The order composable and Firestore write don't change — only the UI that feeds data into them changes.
