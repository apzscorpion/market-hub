# 2.2 — Product Autocomplete

## What We're Building

The type-ahead search inside the order table's product column. As the retailer types, matching products appear in a dropdown. This is the primary way products are selected for an order.

## Component: `ProductAutocomplete.vue`

**File**: `components/retailer/ProductAutocomplete.vue`

### How It Works

```
User types "van"
  ↓
Search against: name, nickname, aliases
  ↓
Matches: "Vanilla Big Pack" (nickname: "Vanilla")
  ↓
Dropdown shows:
  ┌─────────────────────────────┐
  │ Vanilla (Vanilla Big Pack)  │  ← nickname first, full name in gray
  │ ₹250 / box                 │
  └─────────────────────────────┘
  ↓
User selects → row fills: product, unit, price
```

### Search Logic

The autocomplete searches the product list (already loaded in Pinia store from Phase 1).

Search priority:
1. **Exact nickname match** — "vanilla" matches nickname "Vanilla" (highest confidence)
2. **Nickname starts with** — "van" matches "Vanilla"
3. **Alias exact match** — "വാനില" matches alias "വാനില"
4. **Alias starts with** — "വാനി" matches alias "വാനില"
5. **Full name contains** — "big pack" matches "Vanilla Big Pack"

```typescript
function searchProducts(query: string): Product[] {
  const q = query.toLowerCase().trim()
  if (!q) return []

  return products.value
    .filter(p => {
      const nickname = p.nickname.toLowerCase()
      const name = p.name.toLowerCase()
      const aliasMatch = p.aliases.some(a => a.toLowerCase().includes(q))

      return nickname.includes(q) || name.includes(q) || aliasMatch
    })
    .sort((a, b) => {
      // Prioritize nickname matches over full name matches
      const aScore = a.nickname.toLowerCase().startsWith(q) ? 0 : 1
      const bScore = b.nickname.toLowerCase().startsWith(q) ? 0 : 1
      return aScore - bScore
    })
    .slice(0, 8) // Max 8 suggestions
}
```

Each dropdown item shows:
- **Nickname** (bold) + full name (gray, smaller)
- Price + unit type

### Keyboard Behavior

| Key | Action |
|-----|--------|
| Type characters | Filter products, show dropdown |
| Arrow Down | Highlight next suggestion |
| Arrow Up | Highlight previous suggestion |
| Enter | Select highlighted suggestion |
| Escape | Close dropdown, keep text |
| Tab | Select highlighted suggestion AND move to qty cell |

### On Product Selected

When a product is selected from the dropdown:

1. Set `row.productId` = product.id
2. Set `row.productName` = product.name
3. Set `row.nickname` = product.nickname
4. Set `row.unitType` = product.unitType
5. Set `row.price` = product.price
6. Set `row.quantity` = 1 (if not already set)
7. Recalculate `row.total`
8. Move focus to the quantity cell

### Duplicate Product Handling

If the user selects a product that's already in another row:
- **Option A**: Highlight the existing row and increment its quantity (preferred)
- **Option B**: Allow duplicates but show a warning

Go with Option A for better UX. When the user selects "Vanilla" and it's already in row 2, focus jumps to row 2 and increments the quantity.

### Empty State

If no products match the search query:
- Show "No products found for '[query]'"
- If the query looks like a number (e.g., "5"), don't search — the user probably tabbed into the wrong cell

## Implementation Details

The autocomplete component needs:
- `v-model` for the text input value
- `@select` event when a product is chosen
- `@clear` event when the input is emptied
- Ref-based focus management (to programmatically focus input)
- Click outside to close dropdown
- Debounce the search (100ms is enough for client-side)

Use `Teleport` for the dropdown if z-index issues arise with the table.
