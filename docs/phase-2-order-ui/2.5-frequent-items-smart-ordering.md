# 2.5 — Frequent Items & Smart Ordering

## What We're Building

Intelligence that makes ordering faster over time. The system learns what each retailer orders most and surfaces those products first — above the search bar, above the product catalog, above everything.

## Why This Matters

A retailer who orders 30 products typically reorders the same 15-20 items every time. Making them search or scroll through the full catalog every time is wasted effort. Show them what they actually buy.

## How Frequent Items Are Computed

Data source: `retailerStats/{retailerId}.frequentProducts`

This is pre-computed when orders are placed/delivered:

```typescript
interface FrequentProduct {
  productId: string
  productName: string
  nickname: string
  totalQuantity: number   // cumulative across all orders
  orderCount: number      // how many separate orders included this
  lastOrderedAt: Date
}
```

Sorted by `orderCount` (how often, not how many). A product ordered in 20 out of 25 orders is more "frequent" than one ordered once but in quantity 500.

### Max Items Shown

Show top 8-10 frequent items. More than that clutters the UI. If the retailer hasn't ordered enough to have frequent items, hide this section entirely.

Threshold: Show after the retailer has placed at least 3 orders.

## Smart Autocomplete Boost

The product autocomplete (from 2.2) should also use frequency data to rank results:

When searching "van", if the retailer frequently orders "Vanilla Big Pack", it should appear ABOVE "Vanilla Cup Small" even if both are alphabetically equal.

```typescript
function searchWithFrequencyBoost(
  query: string,
  products: Product[],
  frequentProductIds: string[]
): MatchResult[] {
  const results = matchProducts(query, products) // base matching from 2.3

  // Boost frequent products
  return results.sort((a, b) => {
    const aFrequent = frequentProductIds.includes(a.product.id) ? -1 : 0
    const bFrequent = frequentProductIds.includes(b.product.id) ? -1 : 0
    // Frequency boost + score ordering
    return (a.score + aFrequent) - (b.score + bFrequent)
  })
}
```
